# Product Concept

## コンセプト

meet-jerky は macOS 26+ 向けの、あらゆるオンライン会議を自動検知し、マイク音声とデスクトップ音声を別トラックで取得して、リアルタイムに高精度文字起こし・履歴化・AI 議事録化する Mac ネイティブアプリである。

Google Meet / Zoom / Microsoft Teams などの会議を、ブラウザ URL、アプリ名、ウィンドウ情報、アクティブアプリ、音声状態から検知する。

会議中は、マイク入力を「自分の音声」、デスクトップ/アプリ音声を「相手側の音声」として別々に取得し、それぞれをリアルタイム文字起こしする。結果はタイムスタンプで統合し、画面上の邪魔にならないウィンドウで確認できる。

会議後は、文字起こし履歴をアプリ内で検索・閲覧でき、AI連携によって要約、議事録、決定事項、ToDo、話者別整理、辞書補正を行える。

## 品質基準

- 高速: 会議中に実用できる低遅延を最優先する。
- 高精度: 専門用語、人名、社内用語に強くする。
- 自動化: ユーザーが録音の開始を忘れても、検知・通知・開始できる体験を目指す。
- 安心感: 録音・文字起こし中であること、保存範囲、AI 送信有無を明確にする。
- Mac らしさ: 軽量、控えめ、洗練された UI にする。Claude / Notion / ChatGPT のように情報密度が高く、邪魔にならない体験を目指す。

## 対応対象

- Google Meet
- Zoom
- Microsoft Teams
- ブラウザベースの会議 URL
- デスクトップアプリとして動作する会議アプリ

## 音声取得方針

自分の音声はマイク入力から取得する。

相手側の音声はデスクトップ音声またはアプリ音声から取得する。

「自分」と「相手側全体」を別トラックとして安定して扱うことを重視する。相手側の複数参加者を個別に完全分離することは、話者分離や会議サービス連携を含む別課題として扱う。

## 機能領域

- 会議検知
- 通知
- マイク音声取得
- デスクトップ/アプリ音声取得
- リアルタイム文字起こし
- フローティングウィンドウ表示
- 履歴保存
- 履歴検索
- AI 議事録生成
- 決定事項とToDo抽出
- 辞書登録
- 辞書による文字起こし補正または後処理補正

## 非目標

- 録音・文字起こしのステルス動作はしない。
- 課金が必要な API やサービスには勝手に依存しない。
- 相手側の複数参加者を常に完全分離できることを前提にしない。
- macOS の権限表示や録音状態の透明性を犠牲にしない。
