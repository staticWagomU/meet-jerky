## handoff 起動の制約 (重要)
- `claude-agent-handoff-main.sh` は **interactive TUI 起動のため stdin redirect 化されていない** (line 39-41 rustdoc コメント = 引数渡しで TUI keep open が watchdog inspect 要件)
- そのため handoff prompt は依然として **tmux 引数長制限の影響を受ける** (4209 bytes でも too long エラー precedent あり)
- 対策 = 「ファイル参照型 handoff prompt」**precedent を継承**: 起動 prompt は短縮 + フル handoff 内容は別ファイル (`docs/handoff/mjc-main-YYYYMMDD-N.txt`) に書き、起動 prompt 内で「Read で完全に読め」と指示する (本 prompt と同パターン、連続 27 セッション目)

## worker prompt 必須要素 (85 連続ループで実証済、継続適用、7 つ全て明示)

worker prompt には以下を **必ず** 含める:
1. **冒頭で「AGENT_LOG.md の末尾 350 行を読め」**
2. **「AGENT_LOG.md は時系列順 = 最古ログが先頭、最新ログが末尾。新規追記は必ずファイル末尾に行う」**
3. **「先頭は絶対に触らない」**
4. **具体手順**: `tail -10 AGENT_LOG.md` で末尾を確認 → 末尾の `---` 直後に追記
5. **規模超過防止段落** = 担当範囲外の編集禁止 + test 件数の上限明示
6. **大型ファイルは Read 全体禁止 = tail/head/grep で対象範囲のみ参照**
7. **「行末の空白文字 (trailing whitespace) 禁止」**

**追加推奨 (Loop 60-61 経験から)**:
- worker prompt に「commit hash 埋め込み手順」を **明示的に** 記載 (2-commit pattern: refactor commit → hash 取得 → AGENT_LOG.md 更新 → chore commit)、または「`<COMMIT_HASH>` placeholder を使い後で sed 置換 + amend する」手順、いずれかを worker が選べるよう許容
- Loop 60-61 では worker が実態確認 + 自律 chore 修正で適応したが、明示記載で初期から正確に動作する想定
