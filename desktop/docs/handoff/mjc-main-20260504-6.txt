## 直前のセッションでの判断履歴と注意点

### 確立されたパターン (本セッションで定着・累積)
- **HTTP status 境界網羅** (本セッション ループ 1): `match status { 500..=599 => ... }` の range 両端 (499/600) を test し、5xx upper/lower bound の誤リファクタを即座に検知。`200` (success) を error path に誤渡された場合も Other で safe 分類されることを契約として固定。
- **multibyte char base 切り詰め検知装置** (本セッション ループ 1): `"あ".repeat(N)` で `.bytes().take()` への誤リファクタが invalid UTF-8 panic に至るバグを `starts_with("あ")` で検知。`MAX_ERROR_BODY_CHARS` の char base 切り詰め契約を保護。
- **case-sensitive 仕様の文書化** (本セッション ループ 2): `match language { "ja" => ..., _ => ... }` の case-sensitive 仕様を「大文字/mixed-case → fallback」の 4 ケースで test。case-insensitive 化リファクタの意図明示化と上位層の正規化責任の境界を明示。
- **3 段 fallback 優先順位の網羅** (本セッション ループ 3): `extract_transcript` (transcript→text→words[]) と `extract_error_message` (message→error:string→error.message:object) を共存ケース + 個別 fallback ケースで網羅。`Some("")` vs `None` の意味区別 (field 存在 vs 不在) を明示。
- **f64 round() 識別パターン** (本セッション ループ 4): `0.4999 / 0.5001 / 0.5` の 3 値で round vs floor vs ceil vs trunc をすべて検知できる秀逸な fractional 設計。
- **cross-function error 文言 invariant** (本セッション ループ 5): `assert_eq!(err1, err2)` で 2 つの fn が同じ error 形式を返す不変条件を 1 行で固定。片方の文言だけが変わった時の即座検知装置。
- **UTC offset 多様性 (negative offset 繰り下がり)** (本セッション ループ 5): JST midnight cross と UTC-5 midnight 繰り下がりの対称テストで、offset 計算が正方向のみ動く誤実装を検知。

### 累積パターン (前セッションから継承)
- **テスト追加方針**: private fn の **直接呼び出しテスト**で境界仕様と防御コードの存在意義を裏付ける。間接テストとの重複を許容しつつ、関数分割リファクタリングへの耐性を獲得。
- **エラー文言の「contract enforcement」パターン**: 実装と test に同じ `&'static str` を書くことで、ユーザー向け文言を不用意に変えると CI が落ちて気付ける契約強制。
- **境界値テストの「対称ペア」**: 同じ境界 (例: `MAX_JS_DATE_UNIX_SECS`) を 2 層の責任 (parser vs save wrapper) で観測。
- **3 層テスト構造** (前セッション ループ 6): 共通実装 + wrapper 経由 = 18 件で同じ仕様を 3 視点から守る (audio_utils 例)。
- **OS 互換 ErrorKind 契約強制**: `is_err()` のみだとエラー種別が pinned されない。`unwrap_err().kind() == ErrorKind::NotFound` に補強することで OS 文言差異を吸収しつつ type-level の不変条件を守る。
- **state machine 混在遷移テスト**: 同じ guard logic を持つ複数 entry point について「片方が active な状態で他方を呼ぶ」遷移を直接 test することで、guard 統合/分離リファクタへの耐性を獲得。
- **format prefix の層内独立確認**: id format `"{started_at}-{seq}"` の前半を session.rs 側で直接 test。session_store のパーサーに依存せず、format 引数順序の入れ替えを session 側で即座に検知。
- **エスケープ漏れの網羅補完**: エスケープ対象 6 文字すべてを個別 1 文字 + escape 確認で網羅。
- **UI 文言一致の契約強制**: 2 つの API の error 文言が完全一致 (`"API キーが空です"`) であることを `assert_eq!(a, b)` で確認 → 体験統一性の不変条件。
