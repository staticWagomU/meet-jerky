# 背景
`classify_meeting_url` は Google Meet / Zoom / Microsoft Teams の URL を分類する純粋関数で、URL の prefix・port・path・query を厳密にチェックして誤検知を防いでいる。既存の test は会議サービス本体・トレーリングドット・whitespace 混入などをカバーしているが、以下の境界仕様が明示テスト未整備で、回帰の温床になり得る。本タスクではこれら **既存仕様を裏付けるテスト** を追加するのみで、実装ロジックは一切変更しない。

# 仕様: 追加するテスト一覧

すべて `mod tests` 内に新しい `#[test]` 関数として追加すること。既存の test は変更しない。各 test 関数名は説明的に。

## 1. `classify_meeting_url_rejects_empty_and_whitespace_only`
- `""` (空文字列) → None
- `"   "` (半角スペースのみ) → None
- `"\t\n"` (タブ + 改行のみ) → None
- `"\u{3000}"` (全角スペース 1 文字) → None  ※ trim().chars().any(is_whitespace) で reject される

## 2. `classify_meeting_url_rejects_non_http_schemes`
以下すべて None:
- `"file:///path/to/file"`
- `"ftp://meet.google.com/abc-defg-hij"`
- `"ws://meet.google.com/abc-defg-hij"`
- `"chrome://settings/"`
- `"javascript:alert(1)"`  ※ `://` がないので split_once で None になる経路

## 3. `classify_meeting_url_rejects_missing_scheme`
- `"meet.google.com/abc-defg-hij"` → None
- `"//meet.google.com/abc-defg-hij"` → None
- `"/abc-defg-hij"` → None

## 4. `classify_meeting_url_rejects_userinfo_in_authority`
host poisoning 防御の確認:
- `"https://attacker@meet.google.com/abc-defg-hij"` → None
- `"https://user:pass@meet.google.com/abc-defg-hij"` → None
- `"https://meet.google.com@evil.example.com/abc-defg-hij"` → None  (これも `@` を含むため reject)

## 5. `classify_meeting_url_rejects_invalid_port`
- `"https://meet.google.com:/abc-defg-hij"` (空ポート) → None
- `"https://meet.google.com:abc/abc-defg-hij"` (非数値ポート) → None
- `"https://meet.google.com:99999/abc-defg-hij"` (u16 範囲外) → None
- `"https://meet.google.com:65536/abc-defg-hij"` (u16 上限超過) → None
- `"https://meet.google.com:-1/abc-defg-hij"` (負数) → None

## 6. `classify_meeting_url_accepts_valid_port`
肯定ケースの確認 (回帰防止):
- `"https://meet.google.com:8443/abc-defg-hij"` → Some (`service: "Google Meet"`, `host: "meet.google.com"`)
- `"https://meet.google.com:0/abc-defg-hij"` → Some (port 0 は u16 で valid)

ヒント: 既存の `MeetingUrlClassification` フィールドへの assertion は、L811-862 付近の既存 test (`classify_meeting_url_returns_service_and_host_only` など) のスタイルに合わせる。`Some(MeetingUrlClassification { service: ..., host: ... })` リテラル比較。

## 7. `classify_meeting_url_rejects_ipv6_host_for_meeting_services`
ブラケット形式 IPv6 は parser 側でサポートされているが、Google Meet / Zoom / Teams のホスト判定はドメイン名ベースなので必ず None になる:
- `"https://[::1]/j/123456789"` → None
- `"https://[2001:db8::1]/abc-defg-hij"` → None
- `"https://[::1]:8443/l/meetup-join/secret"` → None

## 8. `classify_meeting_url_rejects_double_scheme_separator`
- `"https://://meet.google.com/abc-defg-hij"` → None  (split_once により scheme が "https" → after_scheme が "://meet.google.com..." → authority が空文字 → reject)
- `"https://"` → None (authority 空)
- `"https:///abc-defg-hij"` → None (authority 空)
