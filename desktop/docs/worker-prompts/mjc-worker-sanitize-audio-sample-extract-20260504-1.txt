## 1. `src-tauri/src/audio_utils.rs` を新規作成

内容 (この通り 1:1 でファイル全体):

```rust
//! 音声サンプル処理の共通ユーティリティ。
//!
//! `audio.rs` と `system_audio.rs` で重複していた sanitize 処理を集約する。

/// 音声サンプル (f32) を正規化する。
///
/// - `NaN` / `±Inf` は 0.0 に置換 (downstream の RMS 計算で NaN 伝播を防ぐ)
/// - 有限値は `[-1.0, 1.0]` の範囲にクランプ
pub fn sanitize_audio_sample(sample: f32) -> f32 {
    if sample.is_finite() {
        sample.clamp(-1.0, 1.0)
    } else {
        0.0
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn sanitize_audio_sample_returns_zero_for_nan() {
        assert_eq!(sanitize_audio_sample(f32::NAN), 0.0);
    }

    #[test]
    fn sanitize_audio_sample_returns_zero_for_positive_infinity() {
        assert_eq!(sanitize_audio_sample(f32::INFINITY), 0.0);
    }

    #[test]
    fn sanitize_audio_sample_returns_zero_for_negative_infinity() {
        assert_eq!(sanitize_audio_sample(f32::NEG_INFINITY), 0.0);
    }

    #[test]
    fn sanitize_audio_sample_clamps_above_one_to_one() {
        assert_eq!(sanitize_audio_sample(1.5), 1.0);
    }

    #[test]
    fn sanitize_audio_sample_clamps_below_minus_one_to_minus_one() {
        assert_eq!(sanitize_audio_sample(-1.5), -1.0);
    }

    #[test]
    fn sanitize_audio_sample_passes_through_finite_values_in_range() {
        assert_eq!(sanitize_audio_sample(0.0), 0.0);
        assert_eq!(sanitize_audio_sample(0.5), 0.5);
        assert_eq!(sanitize_audio_sample(-0.5), -0.5);
        assert_eq!(sanitize_audio_sample(1.0), 1.0);
        assert_eq!(sanitize_audio_sample(-1.0), -1.0);
    }
}
```

# 設計上の注意点

- `audio.rs` と `system_audio.rs` の wrapper fn は **削除しない**。理由: モジュール境界での命名 (`sanitize_sample` / `sanitize_pcm_sample`) を残すことで、各モジュール内の呼び出し箇所を変える必要がない。「共通化のためにモジュール内 API を不必要に変更する」のは Tidy First の最小スライス原則に反する。
- 既存テストは「`audio.rs::sanitize_sample` 経由で committed の挙動が動く」「`system_audio.rs::sanitize_pcm_sample` 経由で committed の挙動が動く」を引き続き保証する (委譲経由でも同じテストが機能)。
- `audio_utils.rs` 自体の独自テスト 6 件は「共通実装を直接観測する」契約強制。3 層の test (audio.rs / system_audio.rs / audio_utils.rs) で同じ仕様を異なる視点で裏付ける構造。
- `pub fn sanitize_audio_sample` は crate-public。`pub(crate)` でもよいが、将来別モジュールから使う可能性を考慮し `pub` で十分 (lib crate なので外部 binary には公開されない)。
- 新規 dependency 追加なし。`use` 追加なし (`crate::audio_utils::sanitize_audio_sample` を fully qualified で書くため)。
