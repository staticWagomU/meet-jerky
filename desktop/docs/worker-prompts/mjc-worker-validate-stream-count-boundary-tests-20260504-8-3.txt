## 1. 背景 / Why
`validate_stream_count_for_engine` (line 741-756) は Apple Speech のクラッシュ予防のための安全弁:

```rust
fn validate_stream_count_for_engine(
    engine_type: &crate::settings::TranscriptionEngineType,
    stream_count: usize,
) -> Result<(), String> {
    if matches!(
        engine_type,
        crate::settings::TranscriptionEngineType::AppleSpeech
    ) && stream_count > 1
    {
        return Err(
            "Apple SpeechAnalyzer は現在、マイクと相手側音声の同時文字起こしを安全に処理できません。クラッシュを防ぐため、どちらか片方の音声ソースだけで開始するか、Whisper / OpenAI Realtime / ElevenLabs Realtime を選択してください。"
                .to_string(),
        );
    }
    Ok(())
}
```

既存テスト 3 件:
- `test_apple_speech_rejects_multiple_available_streams` (line 1421-1430): stream_count=2 で Err、`contains("Apple SpeechAnalyzer")` と `contains("同時文字起こし")` の **部分一致のみ**
- `test_apple_speech_allows_single_available_stream` (line 1432-1436): stream_count=1 で Ok
- `test_other_engines_allow_multiple_available_streams` (line 1438-1448): Whisper/OpenAIRealtime/ElevenLabsRealtime の stream_count=2 で Ok

**Gap**:
- エラー文言の **完全一致** assert がない (UI 文言の不用意な変更が検知されない)
- boundary の `stream_count > 1` の **片側 (= 1)** だけしか確認していない。3 streams や usize::MAX の boundary 確認がない
- Apple Speech の **0 streams** 入力 (= 1 streams 同等の Ok 経路) が確認されていない
- 他 engine の **0 streams / usize::MAX streams** が確認されていない (3 engines × 2 stream values の boundary)

これらを 5 件の direct test で固定する。クラッシュ防止の安全弁の文言と挙動が「不変条件」として保護される。

## 2. 具体的 test 案 (assertion レベル)

`super::*` は line 1241 付近で既に use 済み (既存テストと同じ流儀)。`crate::settings::TranscriptionEngineType::*` 経由でアクセスする。

### T1. `validate_stream_count_for_engine_apple_speech_rejects_two_with_exact_error_message`
```rust
let err = validate_stream_count_for_engine(
    &crate::settings::TranscriptionEngineType::AppleSpeech,
    2,
)
.unwrap_err();
assert_eq!(
    err,
    "Apple SpeechAnalyzer は現在、マイクと相手側音声の同時文字起こしを安全に処理できません。クラッシュを防ぐため、どちらか片方の音声ソースだけで開始するか、Whisper / OpenAI Realtime / ElevenLabs Realtime を選択してください。",
    "クラッシュ防止の UI 文言を完全一致で固定 (UI/log 文言契約)"
);
```
**意図**: 既存 `contains` 部分一致では検知できない文言の typo / 句読点変更 / 訳文ずれを完全一致で検知。**句点は全角「。」、カンマは全角「、」、スラッシュは半角「/」、半角スペース込み**を実装からコピー必須。

### T2. `validate_stream_count_for_engine_apple_speech_rejects_three_streams_with_same_error_message`
```rust
let err = validate_stream_count_for_engine(
    &crate::settings::TranscriptionEngineType::AppleSpeech,
    3,
)
.unwrap_err();
assert_eq!(
    err,
    "Apple SpeechAnalyzer は現在、マイクと相手側音声の同時文字起こしを安全に処理できません。クラッシュを防ぐため、どちらか片方の音声ソースだけで開始するか、Whisper / OpenAI Realtime / ElevenLabs Realtime を選択してください。",
    "stream_count=3 でも 2 と同じエラー文言で reject される (`stream_count > 1` の boundary 挙動)"
);
```
**意図**: `stream_count > 1` の boundary 上で 2 と 3 が「同じエラー」を返す現契約を固定。将来「3 streams 以上は別エラー」のような分岐リファクタを入れたらこの test で検知。

### T3. `validate_stream_count_for_engine_apple_speech_rejects_usize_max_streams`
```rust
let err = validate_stream_count_for_engine(
    &crate::settings::TranscriptionEngineType::AppleSpeech,
    usize::MAX,
)
.unwrap_err();
assert!(
    err.contains("Apple SpeechAnalyzer"),
    "usize::MAX boundary でも reject される (overflow ガードなしの現契約)"
);
```
**意図**: usize::MAX boundary でも overflow なく `stream_count > 1` 条件が成立し reject されることを固定。`assert!` の contains にしているのは、エラー文言完全一致は T1/T2 で十分カバー済みのため (assert 数の重複を避ける)。

### T4. `validate_stream_count_for_engine_apple_speech_allows_zero_streams`
```rust
validate_stream_count_for_engine(
    &crate::settings::TranscriptionEngineType::AppleSpeech,
    0,
)
.expect("stream_count=0 は `> 1` 条件不成立で Apple Speech でも Ok を返す現契約");
```
**意図**: boundary 下限 (0 streams) が Ok を返す現契約を固定。実用上は 0 streams で transcription を始めることはないが、validation fn としての契約 (`stream_count > 1` のみが reject 条件) を明示。将来「Apple Speech は 0 streams も reject」のような防御強化を入れたらこの test で検知。

### T5. `validate_stream_count_for_engine_other_engines_allow_zero_and_usize_max_streams`
```rust
for engine in [
    crate::settings::TranscriptionEngineType::Whisper,
    crate::settings::TranscriptionEngineType::OpenAIRealtime,
    crate::settings::TranscriptionEngineType::ElevenLabsRealtime,
] {
    validate_stream_count_for_engine(&engine, 0)
        .expect("Apple Speech 以外は 0 streams でも Ok を返す現契約");
    validate_stream_count_for_engine(&engine, usize::MAX)
        .expect("Apple Speech 以外は usize::MAX streams でも Ok を返す現契約");
}
```
**意図**: 既存 `test_other_engines_allow_multiple_available_streams` (stream_count=2 のみ) では確認されていない 0 と usize::MAX boundary を 3 engine × 2 boundary = 6 assertion で固定。「Apple Speech 以外は無条件 Ok」の現契約を boundary 全範囲で保護。
