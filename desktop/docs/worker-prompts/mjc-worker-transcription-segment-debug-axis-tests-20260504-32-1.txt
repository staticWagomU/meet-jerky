## 指示内容: 「Debug 軸補強」パターン 5 連続 application = TranscriptionSegment

`TranscriptionSegment` は `src-tauri/src/transcription.rs` l.27-39 で `#[derive(Debug, Clone, Serialize)]` + `#[serde(rename_all = "camelCase")]` で定義された struct。6 fields:
- `text: String`
- `start_ms: i64`
- `end_ms: i64`
- `source: Option<TranscriptionSource>` + `#[serde(skip_serializing_if = "Option::is_none")]`
- `speaker: Option<String>` + `#[serde(skip_serializing_if = "Option::is_none")]`
- `is_error: Option<bool>` + `#[serde(skip_serializing_if = "Option::is_none")]`

`TranscriptionSource` は同ファイル l.19-23 で `#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize)]` + `#[serde(rename_all = "snake_case")]` で定義された enum (Microphone / SystemAudio)。

以下 3 つの test を `mod tests` 末尾 (= 既存最終 test `transcription_error_payload_partial_eq_holds_reflexive_and_differs_for_distinct_error_or_source` 直後の `}` の **直前**) に追加せよ。

### T1: `transcription_segment_debug_output_contains_struct_name_all_six_field_names_and_values`
**契約**: TranscriptionSegment の Debug 出力が型名 `TranscriptionSegment` + **全 6 field 名 (snake_case のまま: `text`, `start_ms`, `end_ms`, `source`, `speaker`, `is_error`)** + 各 field 値 + Some/None 両 variant + enum variant 名を含む。
**test 設計**:
- 全 field を埋めた segment を 1 つ作る (text="hello", start_ms=100, end_ms=2000, source=Some(SystemAudio), speaker=Some("自分"), is_error=Some(true)).
- `let dbg = format!("{:?}", segment);` で Debug 出力を取る。
- 以下を `assert!(dbg.contains("..."), "...")` で順次確認:
  - `"TranscriptionSegment"` (型名)
  - `"text"`, `"start_ms"`, `"end_ms"`, `"source"`, `"speaker"`, `"is_error"` (snake_case の全 6 field 名)
  - `"hello"`, `"100"`, `"2000"`, `"自分"`, `"true"` (値)
  - `"Some"` (Option<T> の Some variant)
  - `"SystemAudio"` (TranscriptionSource enum の variant 名)
- これにより Debug 派生が消されたり、`#[derive(Debug)]` を手動実装に置き換えて field を隠蔽する誤改修を遮断する。

### T2: `transcription_segment_debug_output_equals_after_clone_for_some_and_none_variants`
**契約**: `#[derive(Clone)]` 派生で clone した segment の Debug 出力が元と完全一致する。Some を全部埋めた case と None を混ぜた case の 2 通りで確認。
**test 設計**:
- 全 Some 埋め segment (text="full", start_ms=0, end_ms=500, source=Some(Microphone), speaker=Some("自分"), is_error=Some(false)) を作る → `assert_eq!(format!("{:?}", s), format!("{:?}", s.clone()))`
- 全 Option None segment (text="bare", start_ms=-1, end_ms=0, source=None, speaker=None, is_error=None) を作る → 同じく `assert_eq!`
- これにより Clone を手動実装して Option field を加工する誤改修を遮断する。

### T3: `transcription_segment_serde_serializes_with_camel_case_field_names_and_skips_none_options`
**契約**: `#[serde(rename_all = "camelCase")]` + `#[serde(skip_serializing_if = "Option::is_none")]` の組み合わせで、serde JSON 出力では:
- 必須 field 名が camelCase (`text`, `startMs`, `endMs`)
- snake_case (`start_ms`, `end_ms`, `is_error`) は **JSON に出ない**
- None の Option field は JSON から省略される
- Some の Option field の field 名は camelCase (`isError`, `speaker`, `source` (※ source は元から snake と camel の差異なし))
**test 設計**:
- `serde_json::to_value(&segment).unwrap()` で `serde_json::Value::Object` を取得する (or to_string -> contains).
- まず None mix segment (text="bare", start_ms=-5, end_ms=0, source=None, speaker=None, is_error=None) を Serialize:
  - `assert!(json.contains("\"text\""))`
  - `assert!(json.contains("\"startMs\""))` (camelCase)
  - `assert!(json.contains("\"endMs\""))`
  - `assert!(!json.contains("\"start_ms\""))` (snake_case が出ないこと)
  - `assert!(!json.contains("\"end_ms\""))`
  - `assert!(!json.contains("\"is_error\""))`
  - `assert!(!json.contains("\"isError\""))` (None なので出ない)
  - `assert!(!json.contains("\"speaker\""))` (None なので出ない)
  - `assert!(!json.contains("\"source\""))` (None なので出ない)
- 次に全 Some 埋め segment (text="full", start_ms=10, end_ms=20, source=Some(SystemAudio), speaker=Some("相手側"), is_error=Some(true)) を Serialize:
  - `assert!(json.contains("\"isError\""))` (Some なので出る、camelCase)
  - `assert!(!json.contains("\"is_error\""))` (snake_case は出ない)
  - `assert!(json.contains("\"systemAudio\"") || json.contains("system_audio"))` (TranscriptionSource は snake_case の方なので `system_audio` で OK)
  - 注: TranscriptionSource は `#[serde(rename_all = "snake_case")]` なので value は `"system_audio"` 形式 = camelCase 化されない (struct field 名と enum variant value は別ルール)
- `serde_json::to_string(&segment).unwrap()` で String 化して contains で判定する方式が読みやすい。
- これにより serde rename_all が外されたり個別 rename された誤改修、skip_serializing_if が外された誤改修を遮断する。
