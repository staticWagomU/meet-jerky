## 未保護の軸 (本 Loop で 3 軸 application)

### T19 (Debug 出力 = SessionSegment field 名軸)
名称: `session_segment_debug_output_contains_struct_name_and_all_three_field_names`
契約:
- `SessionSegment { speaker: "alice".to_string(), timestamp_offset_secs: 42, text: "hi".to_string() }` の `format!("{:?}", segment)` が以下を全て含むこと:
  - 型名 `"SessionSegment"`
  - field 名 `"speaker"`, `"timestamp_offset_secs"`, `"text"`
  - 値 `"alice"`, `"42"`, `"hi"`
- `#[derive(Debug)]` 自動派生による「struct with named fields は型名+全フィールド名出力」Rust 言語仕様の executable specification 化
- 「将来 Debug を手動実装に切り替えて field を隠蔽する誤改修」「field を rename する誤改修」を遮断する装置

### T20 (Debug 出力 = Session field 名軸)
名称: `session_debug_output_contains_struct_name_and_all_five_field_names_including_options_and_vec`
契約:
- `Session::start("Daily".to_string(), 1234567890)` で生成した Session の `format!("{:?}", session)` が以下を全て含むこと:
  - 型名 `"Session"`
  - field 名 `"id"`, `"title"`, `"started_at"`, `"ended_at"`, `"segments"`
  - 値 `"Daily"`, `"1234567890"`, `"None"` (Option::None の Debug 出力), `"[]"` (空 Vec の Debug 出力) のうちいずれか or 全て
- 5 field の混在 (String / u64 / Option<u64> / Vec<SessionSegment>) でも Debug 出力に全 field 名が含まれる契約を CI 固定
- 「将来 Debug 手動実装で Option / Vec field を隠蔽する誤改修」を遮断する装置

### T21 (Debug 出力 + Clone 派生 = clone 後 Debug 出力同一軸)
名称: `session_segment_debug_output_equals_after_clone_for_all_field_types`
契約:
- 元 SessionSegment と `.clone()` 後の SessionSegment の `format!("{:?}", ...)` が完全一致 (`assert_eq!(format!("{:?}", original), format!("{:?}", cloned))`)
- field 等価性 (既存 T16 = session_segment_clone_preserves_all_three_fields_independently) を超えて、**Debug 出力 format まで含めて完全コピー** であることを保護
- `#[derive(Debug, Clone)]` の組み合わせで「Clone が field 順や Debug 表現まで含めて 100% 同一」契約の executable specification 化
- 「将来 Clone を手動実装で field 順を変える / 値を加工する誤改修」を遮断する装置
