## 1. 新規ファイル `src-tauri/src/audio_event.rs` を作成
内容は以下の最小モジュール:

```rust
use serde_json::json;

pub(crate) fn build_audio_drop_event_payload(source: &str, dropped: usize) -> serde_json::Value {
    json!({ "source": source, "dropped": dropped })
}
```

- 可視性は `pub(crate)`。lib 外からは見えなくてよいが、`audio.rs` と `system_audio.rs` の両方からインポートする必要があるため、private のままでは不可。
- `#[cfg(target_os = "macos")]` ガードは付けない (この helper 自体は platform-agnostic、json 文字列を組み立てるだけ)。
- ファイル末尾に `mod tests` は **置かない**。テストは既存の audio.rs::tests と system_audio.rs::tests に残し、import 経由で同 fn を引き続き検証する。これにより test 数 450 は完全維持される。

# 実装上の注意 (踏みやすい罠)
- **テストを移動・削除しない**。今回の Tidy はあくまで「fn の置き場所を移す」だけ。テスト数 450 を厳守。テスト名の変更も不要。
- **system_audio.rs の `#[cfg(target_os = "macos")]` ガードは削除しない**。helper fn 側は cfg なしで `audio_event.rs` に置くが、call site (line 333 付近) と test 群はもともと macos-only コードの中にあるので、その cfg 構造は触らない。
- **lib.rs の module 順はアルファベット順**。既存が `audio`, `audio_utils` と並んでいるので `audio_event` はその間に置く。`mod audio_event;` は `mod audio;` の直下、`mod audio_utils;` の直上。
- **可視性は `pub(crate)` で十分**。`pub` にすると外部 API 誤公開の検知が緩むので避ける。
- **不要になった `use serde_json::json;` 削除を忘れない**。clippy `unused_imports` で必ず引っかかるので、削除後に clippy が clean になることで確認する。
- **call site は完全な path (`crate::audio_event::build_audio_drop_event_payload`) を推奨**。`use crate::audio_event::build_audio_drop_event_payload;` を使う場合は、各ファイルの top-level に追加し、name shadowing が起きないか念のため目視確認する。
- **rustfmt の影響でフォーマットが少しズレることがあるので必ず `cargo fmt -p meet-jerky` を最後にかける**。
