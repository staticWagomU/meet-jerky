# 背景 / Why

直前セッション (mjc-main-20260504-14) で「audio-drop-count」イベントの 3 段スライス分解 (rust 抽出 → frontend listener → UI 表示) を完了し、product-concept 優先度 9「録音状態の透明性」を user-visible にしました。

ただし、frontend の累積 state (microphoneDropCountTotal / systemAudioDropCountTotal) はアプリ起動中ずっと累積し続け、新しいセッションを始めても 0 に戻りません。これはユーザー体験上、以下の問題があります:

- 新セッション開始時に「前回の drop count が残っている」という違和感
- 累積値が「いつから」のものか不明 (アプリ起動から?)

本ループの目的は、**session start タイミングで drop count を 0 リセットする** ことで「fresh start = 期待通り」の UX を確立することです。STOP タイミングで reset すると stop 直後に履歴が消えてしまい UX 損失なので、START 側で reset する戦略を採用します。listener error (audioDropCountListenerError) は session 境界に依存しない debug 情報なので reset 対象外とします。

これは前ループの自然な polish であり、audio-drop-count chain の最終仕上げに該当します。
