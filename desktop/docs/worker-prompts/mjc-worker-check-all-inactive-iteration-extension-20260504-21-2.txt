# 背景

前 mjc-main-21 Loop 1 (commit cceffa3) で `parse_throttle_key_to_display_name` 純粋関数を追加し、3 経路 (`bundle_id` 単独 / `"browser:..."` / `"window-title:..."`) の throttle_key を display name に変換できる foundation を完成させた。
ただし `check_all_inactive_bundles` は依然 `WATCHED_BUNDLE_IDS` (Zoom/Teams/Teams legacy/FaceTime の 4 件) のみ巡回 = browser 経路で書き込まれた throttle_key (= mjc-main-20 Loop 3, commit 527a2a3 で書き込み開始) は **読まれない** = browser-only セッション (Safari の Google Meet 等) は inactive 通知が発火しない設計乖離が残っている。

本 Loop で iteration を `last_seen_secs.lock().keys()` の **全件巡回** に変更することで、F-Loop5 が機能完結し、Safari の Google Meet 等でも会議終了通知が実機で動く状態に到達する。

# Why (価値)

- product-concept.md「ユーザーが録音の開始を忘れても 検知・通知・開始できる体験」の **終了側体験を browser-only セッションにも拡張** = アプリ経路と同等の終了通知体験
- 「同じ HashMap (`last_seen_secs`) に異なる形式の key が混在する」設計の境界 (= parse 純粋関数) と消費側 (= iteration) を一貫させる
- mjc-main-20 Loop 3 (527a2a3) で foundation を作った browser 経路の last_seen_secs エントリが、本 Loop で **初めて消費される** = データ収集 → データ消費の対称性が runtime で完成

## 2. `check_all_inactive_bundles` の iteration 変更 (line 284-294 付近)

旧:
```rust
fn check_all_inactive_bundles() {
    let state = match STATE.get() {
        Some(s) => s,
        None => return,
    };
    for (bundle_id, app_name) in WATCHED_BUNDLE_IDS {

}
```

新 (lock を即解放してから iterate、display name 解決失敗時は skip):
```rust
fn check_all_inactive_bundles() {
    let state = match STATE.get() {
        Some(s) => s,
        None => return,
    };
    // last_seen_secs MutexGuard を即解放してから iterate (デッドロック回避)。
    // throttle_key は 3 形式の混在 (bundle_id 単独 / "browser:..." / "window-title:...") = parse_throttle_key_to_display_name で表示名に解釈する。
    let throttle_keys: Vec<String> = state
        .last_seen_secs
        .lock()
        .keys()
        .cloned()
        .collect();
    for throttle_key in throttle_keys {
        if let Some(app_name) = parse_throttle_key_to_display_name(&throttle_key) {
            if let Some(elapsed) = check_meeting_inactive_for_bundle(&throttle_key) {
                show_inactive_notification(&state.app_handle, &app_name, elapsed);
            }
        }
    }
}
```

doc コメント (line 280-283 付近) を更新: 「`last_seen_secs` の key 全件を巡回し、`parse_throttle_key_to_display_name` で display name に変換した上で `check_meeting_inactive_for_bundle` を呼ぶ。アプリ経路 (Zoom/Teams/FaceTime) と browser 経路 (Safari 等) の両方が対象」旨を明示。

## 3. `parse_throttle_key_to_display_name` の dead_code 二重 attribute 削除 (line 456 付近)

Loop 1 で追加した:

**判定基準**: `cargo clippy --all-targets -- -D warnings` で警告が出るかどうか。出るなら `#[cfg_attr(...)]` だけ残す、出ないなら両方削除。
