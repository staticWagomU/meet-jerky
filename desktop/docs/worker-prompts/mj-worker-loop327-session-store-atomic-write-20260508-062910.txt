背景:
`write_session_markdown_to` は現在 `render_session_markdown` 後に `fs::write(path, body)` で既存 Markdown を全文上書きしています。これは途中でプロセス停止や書き込み失敗が起きた場合に、既存の履歴 Markdown が truncate 済みの中途半端な内容になるリスクがあります。会議履歴の保存信頼性を上げるため、同一ディレクトリ内の temp file に書いてから `std::fs::rename` で置き換える最小改善を実装してください。

実装内容:
- `write_session_markdown_to(path, session, offset)` を atomic replace 方式に変更してください。
  - 先に `render_session_markdown(session, offset)?` を評価する既存契約は維持する。
  - temp file は target と同じディレクトリに作る。
  - temp file 名は target filename を元にしつつ、process id と時刻などで通常衝突しにくい名前にする。
  - temp file へ全文を書いた後、`std::fs::rename(temp_path, path)` で置き換える。
  - temp write / rename が失敗した場合は可能な範囲で temp file を削除し、元の error を返す。
- macOS/Unix 前提の同一ディレクトリ rename で十分です。Windows 対応や fsync 追加へ広げないでください。
- `save_session_markdown`、`path_for_session`、Markdown format、render 処理、session manager / command 側、依存関係は変更しないでください。
- 既存の missing parent、render error、完全上書き、direct/save content 一致の契約を維持してください。

テスト:
- 既存 `write_session_markdown_to_truncates_and_overwrites_existing_file_completely` は新実装でも通るよう維持してください。
- 可能なら次の unit test を `session_store.rs` に追加してください。
