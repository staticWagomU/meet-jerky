# 追加する test 3 件 (Tidy First、ユーザー観測不可、振る舞い不変)

## test 1: `finalize_accepts_ended_at_equal_to_started_at_for_zero_duration_session`
- **目的**: SessionManager::finalize が ended_at == started_at (zero-duration session、即座 finalize) を受け入れる契約を CI 固定
- **手順**:
  1. `let manager = SessionManager::new();`
  2. `manager.start("instant_session".into(), 1_700_000_000).expect("start");`
  3. `let session = manager.finalize(1_700_000_000).expect("finalize with ended_at == started_at should succeed");`
  4. `assert_eq!(session.ended_at, Some(1_700_000_000), "zero-duration session の ended_at は started_at と同値");`
  5. `assert_eq!(session.duration_secs(), Some(0), "ended_at == started_at のとき duration は 0");`
  6. `assert!(!manager.is_active(), "finalize 後は非活性");`
- **意図**: SessionManager 層で `if ended_at <= started_at { error }` 等の防衛的 ordering validation を追加する誤改修を遮断する装置。zero-duration session (例: 即座のキャンセル) は real flow で発生しうる。
- **assert メッセージ**: 「ended_at == started_at は valid な finalize 契約。SessionManager 層への ordering validation 追加を CI で遮断する。」

## test 2: `finalize_accepts_ended_at_less_than_started_at_when_clock_goes_backwards`
- **目的**: SessionManager::finalize が ended_at < started_at (clock 逆転、NTP 補正等) を error なしで受け入れ、ended_at を原値透過で保存する契約を CI 固定
- **手順**:
  1. `let manager = SessionManager::new();`
  2. `manager.start("clock_backward_session".into(), 1_700_000_500).expect("start");`
  3. `let session = manager.finalize(1_700_000_000).expect("finalize with ended_at < started_at should succeed (NTP correction tolerance)");`
  4. `assert_eq!(session.ended_at, Some(1_700_000_000), "clock 逆転時も ended_at は原値透過で保存される");`
  5. `assert_eq!(session.duration_secs(), Some(0), "Session::duration_secs は saturating_sub で 0 に飽和する");`
- **意図**: 「if ended_at < started_at { error }」のような防衛的 ordering validation 追加への誤改修を遮断する装置。NTP 補正や夏時間調整で稀に発生する clock 逆転を error にしない契約を保護。Session 層では既存 `duration_secs_saturates_when_end_precedes_start` が duration 飽和を保護しているが、SessionManager 層 (persistence + lock 包装) では未保護。
- **assert メッセージ**: 「clock 逆転 (ended_at < started_at) も valid な finalize 契約。SessionManager 層は ordering validation を行わず原値透過で保存する。NTP 補正への耐性を CI で固定する。」

## test 3: `finalize_preserves_u64_max_ended_at_without_truncation`
- **目的**: SessionManager::finalize が ended_at=u64::MAX (極端値) を truncation なく原値透過で Session.ended_at に保存する契約を CI 固定
- **手順**:
  1. `let manager = SessionManager::new();`
  2. `manager.start("far_future_session".into(), 1_700_000_000).expect("start");`
  3. `let session = manager.finalize(u64::MAX).expect("finalize with ended_at=u64::MAX should succeed");`
  4. `assert_eq!(session.ended_at, Some(u64::MAX), "ended_at=u64::MAX は truncation なく Session に保存される");`
- **意図**: 内部で `ended_at as i64` cast 等を入れた場合に signed overflow で負値化するような誤改修を検知する装置。Loop 1 の T3 (started_at u64::MAX) と対称的に、ended_at 側も u64 型契約を CI 固定。
- **assert メッセージ**: 「ended_at=u64::MAX は truncation なく Session に保存される。`as i64` cast 等の符号付き truncation を CI で遮断する。」
