## 実装上の注意

### dead_code allow 戦略
- 新規フィールド `last_seen_secs` は本 Loop では誰も読み書きしないため、コンパイラが unused field 警告を出す
- 既存 `DetectionState` には既に `#[cfg_attr(not(target_os = "macos"), allow(dead_code))]` (line 83) が struct 全体に付いているため、macOS 以外では既に警告抑制されている
- **macOS 上で field unused 警告が出るかを確認する必要あり**: もし出るなら、新規フィールド単独に `#[allow(dead_code)]` を付ける (struct 全体ではなくフィールドピンポイントで)
  - 例: `#[allow(dead_code)] last_seen_secs: Mutex<HashMap<String, u64>>,`
- 出ない場合は attribute 不要

### 命名
- フィールド名: `last_seen_secs` (`should_notify_meeting_inactive` の引数名と同一)
- 既存 `last_seen` (Instant) との対比が明確
- HashMap のキーは既存 `last_seen` と同じ `String` (bundle_id を入れる予定だが、本 Loop では update しないので問題なし)
