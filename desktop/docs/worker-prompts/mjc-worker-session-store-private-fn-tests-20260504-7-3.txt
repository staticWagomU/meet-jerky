## 3. 実装上の注意
- 追加先は `mod tests` の最後の `}` 直前 (line ~547 付近)。新規 inner mod は不要、tests mod 直下に追加。
- `ErrorKind` は `std::io::ErrorKind`。tests mod の `use super::*;` だけだと参照できないので、各 test 関数内で `use std::io::ErrorKind;` を局所 import するか、tests mod 冒頭に追記。**既存の use 文と重複しないよう注意** (既存に `use super::*;`, `use crate::session::Session;`, `use tempfile::tempdir;` がある)。
- T3 の expected error 文字列 `"session started_at is out of i64 range: 18446744073709551615"` は `u64::MAX = 18446744073709551615`。format の "{label} is out of i64 range: {unix_secs}" を実装からコピーして組み立てる。`assert_eq!` で完全一致。
- T5 の `r"\\"` は raw string で `\\` (2 文字 = backslash 2 つ)、これを unescape すると `\\` の `\` の次が `\` で「6 文字のうち 1 つ (backslash)」にマッチして `escaped` (= 2 つ目の `\`) を push、結果 `"\\"` (1 文字 = backslash 1 つ)。Rust の raw string `r"\\"` と通常の `"\\"` は同じ「文字 \ 2 つ」。unescape 結果 `"\\"` は通常文字列で `"\\"` (= 1 つの `\`)。混乱しやすいので一行ずつ確認しながら書くこと。
- T5 の `r"\!"` (raw string) は backslash + `!` の 2 文字。`!` は 6 文字に含まれないので `\!` がそのまま (2 文字) 返る。
- T5 の lone trailing backslash `r"\"` は単一 backslash 1 文字。実装は `if ch == '\\'` 分岐に入り `chars.peek()` が `None` を返すので `else { out.push(ch) }` の `out.push('\\')` で結果 `"\"` (= backslash 1 文字)。
- 5 件すべて test 関数のみ追加。本体ロジック (line 1-198) は **一切変更しない**。
- 既存 25 件のテスト名・本体は **絶対に変更しない**。
