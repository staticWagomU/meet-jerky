## 追加する test 3 件 (命名と意図を厳守)

### T1: `build_audio_drop_event_payload_with_microphone_source_returns_json_with_source_and_dropped_fields`

- 目的: happy path で payload の構造 (`{"source": "microphone", "dropped": 5}`) を CI 固定。今後フィールド名変更や別フォーマット変更を遮断する装置。
- 実装:
  ```rust
  let payload = build_audio_drop_event_payload(AUDIO_SOURCE_MICROPHONE, 5);
  assert_eq!(payload["source"], "microphone");
  assert_eq!(payload["dropped"], 5);
  ```
- assert メッセージは不要 (`assert_eq!` で十分)。

### T2: `build_audio_drop_event_payload_with_empty_source_passes_through_without_normalization`

- 目的: 空文字 source も passthrough (加工しない / 別 fallback を入れない) する現契約を CI 固定。フィールド対称軸補強パターン (mjc-main-28 Loop 2 の format_session_markdown と同型)。
- 実装:
  ```rust
  let payload = build_audio_drop_event_payload("", 0);
  assert_eq!(payload["source"], "");
  assert_eq!(payload["dropped"], 0);
  ```

### T3: `build_audio_drop_event_payload_with_usize_max_dropped_serializes_without_overflow`

- 目的: `usize::MAX` の dropped 値も JSON シリアライズ可能 = overflow しない現契約を CI 固定。型対称軸補強パターン (mjc-main-27 Loop 3 の i64::MAX と同型)。
- 実装:
  ```rust
  let payload = build_audio_drop_event_payload(AUDIO_SOURCE_SYSTEM_AUDIO, usize::MAX);
  assert_eq!(payload["source"], "system_audio");
  assert_eq!(payload["dropped"], usize::MAX);
  ```
