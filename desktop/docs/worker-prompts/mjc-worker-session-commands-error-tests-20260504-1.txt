## 1. `finalize_and_save_session_inner_returns_error_when_output_dir_path_is_a_file`

`output_dir` 引数のパスが既に **ファイル** として存在する場合、`std::fs::create_dir_all` は `AlreadyExists` 系のエラーを返す。エラーメッセージが Japanese prefix を含むことを確認:

```rust
let manager = SessionManager::new();
let dir = tempdir().unwrap();
// 出力先と同名のファイルを先に作る
let blocking_file = dir.path().join("blocking");
std::fs::write(&blocking_file, b"existing content").unwrap();

start_session_inner(&manager, "会議".into(), 100, dir.path(), jst())
    .expect("start should succeed");
manager
    .append("user".to_string(), 5, "hello".to_string())
    .expect("append should succeed");

// blocking_file がファイルなのでディレクトリとして作成できない
let err = finalize_and_save_session_inner(&manager, &blocking_file, 1_713_333_100, jst())
    .expect_err("should error when path is a file not a directory");

assert!(
    err.starts_with("出力ディレクトリの作成に失敗しました"),
    "unexpected error message: {err}"
);
// finalize 後は manager がアイドル状態に戻る (finalize 自体は成功している)
assert!(!manager.is_active());
```

注意: `finalize_and_save_session_inner` は L92 で `manager.finalize` を **先に** 呼んでセッションを取り出してから create_dir_all → save を呼ぶ。よって create_dir_all が失敗してもセッションは既に finalize 済みでアイドル状態になっている (これは既存仕様 — テストでこれを assert する)。

## 2. `list_session_summaries_inner_returns_error_when_path_is_a_file`

`output_dir` がファイル (ディレクトリではない) を指している場合、`session_store::list_session_summaries` は `read_dir` 段階で `NotADirectory` 系エラーを返す。`list_session_summaries_inner` の `output_dir.exists()` チェック (L140) は通過するため、エラーが伝播されてユーザー向けメッセージに包装される:

```rust
let dir = tempdir().unwrap();
let blocking_file = dir.path().join("not_a_dir.txt");
std::fs::write(&blocking_file, b"hello").unwrap();
assert!(blocking_file.exists());
assert!(blocking_file.is_file());

let err = list_session_summaries_inner(&blocking_file)
    .expect_err("should error when path is a file not a directory");

assert!(
    err.starts_with("セッション一覧の取得に失敗しました"),
    "unexpected error message: {err}"
);
```
