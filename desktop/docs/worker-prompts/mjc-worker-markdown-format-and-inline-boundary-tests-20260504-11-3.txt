### T1: format_session_markdown_joins_three_or_more_segments_with_single_newline_between

**目的**: 3 segment 以上の join 順序と区切り (`\n` 1 個) と各行末 `  ` (trailing 2 spaces) と最終行 trailing newline 無しを完全一致 expected で固定。

```rust
#[test]
fn format_session_markdown_joins_three_or_more_segments_with_single_newline_between() {
    let meta = SessionMeta {
        title: "会議".to_string(),
        started_at_display: "2026-04-17 14:30".to_string(),
    };
    let segments = vec![
        SessionSegment {
            speaker: "A".to_string(),
            timestamp_display: "14:30:01".to_string(),
            text: "first".to_string(),
        },
        SessionSegment {
            speaker: "B".to_string(),
            timestamp_display: "14:30:02".to_string(),
            text: "second".to_string(),
        },
        SessionSegment {
            speaker: "C".to_string(),
            timestamp_display: "14:30:03".to_string(),
            text: "third".to_string(),
        },
    ];

    // 各 segment 行末は "  " (trailing 2 spaces = markdown line break)、間は単一 \n、最終行 \n なし
    let expected = "# 会議 - 2026-04-17 14:30\n\n**[14:30:01] A:** first  \n**[14:30:02] B:** second  \n**[14:30:03] C:** third  ";
    assert_eq!(format_session_markdown(&meta, &segments), expected);

    // 順序保証: A → B → C の入力順がそのまま出力に反映される
    let result = format_session_markdown(&meta, &segments);
    let pos_a = result.find("first").unwrap();
    let pos_b = result.find("second").unwrap();
    let pos_c = result.find("third").unwrap();
    assert!(pos_a < pos_b && pos_b < pos_c, "segment 順序が出力に保たれる契約");
}
```

### T2: format_session_markdown_escapes_special_chars_in_started_at_display_and_timestamp_display

**目的**: meta.started_at_display と segment.timestamp_display にも `inline_markdown_text` が適用される対称性を、escape 対象文字を含む display 文字列で固定。誤改修で「display 系は raw のまま挿入する」変更を入れた際の検知装置。

```rust
#[test]
fn format_session_markdown_escapes_special_chars_in_started_at_display_and_timestamp_display() {
    let meta = SessionMeta {
        title: "title".to_string(),
        started_at_display: "*2026*_04_[17]".to_string(),
        // ↑ started_at_display も `inline_markdown_text` を通る契約 (line 28)
    };
    let segments = vec![SessionSegment {
        speaker: "speaker".to_string(),
        timestamp_display: "[14:30:05]".to_string(),
        // ↑ timestamp_display も `inline_markdown_text` を通る契約 (line 41)
        text: "text".to_string(),
    }];

    let result = format_session_markdown(&meta, &segments);

    // started_at_display 内の `*` `_` `[` `]` が escape されている
    assert!(
        result.contains("\\*2026\\*\\_04\\_\\[17\\]"),
        "started_at_display も escape される契約: result=\n{result}"
    );
    // timestamp_display 内の `[` `]` が escape されている
    assert!(
        result.contains("\\[14:30:05\\]"),
        "timestamp_display も escape される契約: result=\n{result}"

}
```

### T3: inline_markdown_text_escapes_each_char_in_consecutive_or_mixed_special_runs

**目的**: 連続する special chars (`***`, `[[[]]]`, `\\\`) と mixed (`*_*`) でも各文字が独立に escape されることを 4 ケースで固定。`for ch in ...` の独立適用 (line 54-58) の境界。

```rust
#[test]
fn inline_markdown_text_escapes_each_char_in_consecutive_or_mixed_special_runs() {
    // 3 個連続 (asterisk)
    assert_eq!(super::inline_markdown_text("***"), "\\*\\*\\*");
    // 4 個連続 (square bracket open + close)
    assert_eq!(super::inline_markdown_text("[[]]"), "\\[\\[\\]\\]");
    // 3 個連続 (backslash)
    assert_eq!(super::inline_markdown_text("\\\\\\"), "\\\\\\\\\\\\");
    // mixed: `*_*` → `\*\_\*`
    assert_eq!(super::inline_markdown_text("*_*"), "\\*\\_\\*");
    // mixed: `[`code`]` → `\[\`code\`\]`
    assert_eq!(super::inline_markdown_text("[`code`]"), "\\[\\`code\\`\\]");
}
```

### T4: inline_markdown_text_passes_through_emoji_combining_and_zwj_chars_without_escape

**目的**: escape 対象 6 文字 (`\\` `` ` `` `*` `_` `[` `]`) 以外の Unicode は passthrough されることを 4 ケースで固定。emoji / 結合文字 / ZWJ family / 全角記号 (`！`「」※ など) はそのまま。誤改修で「すべての非 ASCII を escape する」変更を入れた際の検知装置。

```rust
#[test]
fn inline_markdown_text_passes_through_emoji_combining_and_zwj_chars_without_escape() {
    // emoji (single code point)
    assert_eq!(super::inline_markdown_text("🎉"), "🎉");
    // emoji + text
    assert_eq!(super::inline_markdown_text("hello🎉world"), "hello🎉world");
    // 結合文字 (基底 'e' + U+0301 結合アキュート)
    let combining = "e\u{0301}"; // = "é"
    assert_eq!(super::inline_markdown_text(combining), combining);
    // ZWJ family emoji (U+1F468 + ZWJ + U+1F469 + ZWJ + U+1F467 + ZWJ + U+1F466)
    let zwj_family = "\u{1F468}\u{200D}\u{1F469}\u{200D}\u{1F467}\u{200D}\u{1F466}";
    assert_eq!(super::inline_markdown_text(zwj_family), zwj_family);
    // 全角記号 (escape 対象 6 文字に含まれない)
    assert_eq!(super::inline_markdown_text("「重要」※"), "「重要」※");
}
```

### T5: inline_markdown_text_treats_full_width_space_and_nbsp_as_split_whitespace

**目的**: `str::split_whitespace` (line 52) は `char::is_whitespace` で分割するため、全角空白 (U+3000) や NBSP (U+00A0) も区切りとして扱われる現契約を 4 ケースで固定。前 mjc-main-9 の `7cc7f94` が transcript_bridge.rs で固定した str::trim 版と対称な、split_whitespace 版の文書化。誤改修で「ASCII space のみで分割する」変更を入れた際の検知装置。

注意: `inline_markdown_text` は `split_whitespace().collect::<Vec<_>>().join(" ")` で正規化する (line 52)。区切られた token 間は ASCII space 1 個に統一される。

```rust
#[test]
fn inline_markdown_text_treats_full_width_space_and_nbsp_as_split_whitespace() {
    // U+3000 全角空白で分割される (token 間は ASCII space 1 個に正規化)
    assert_eq!(super::inline_markdown_text("a　b"), "a b");
    // U+00A0 NBSP も分割される
    assert_eq!(super::inline_markdown_text("a\u{00A0}b"), "a b");
    // U+2028 LINE SEPARATOR も分割される
    assert_eq!(super::inline_markdown_text("a\u{2028}b"), "a b");
    // 混在: ASCII + NBSP + 全角空白 + tab → 全て区切られて単一 ASCII space で join
    assert_eq!(super::inline_markdown_text("x \u{00A0}\u{3000}\ty"), "x y");
    // 全角空白のみ (whitespace_only と同じく empty に正規化)
    assert_eq!(super::inline_markdown_text("　　　"), "");
}
```

## 3. 実装上の注意
----------------------------------------------------------------------

- すべての test は **既存 `mod tests` (line 67-186) の末尾** に追加する。`mod tests` の中括弧の閉じ位置 (line 186) に注意。
- 既存の `use super::*;` をそのまま流用。`super::inline_markdown_text` の呼び方も既存パターン (line 144 等)。
- T1 の expected 文字列は **改行と trailing 2-spaces** の正確な再現が必要。`"first  \n**[..."` (2 spaces + \n) で書く。raw string (`r#"..."#`) は trailing space を視覚化しにくいので普通の `"..."` で OK、ただし `\n` は escape 必要。
- T2 の `inline_markdown_text` は order 保存型 escape なので `"\\*2026\\*\\_04\\_\\[17\\]"` のように **入力 char 順** に escape を挟む。
- T3 の backslash 3 個 `"\\\\\\"` は Rust 文字列 literal で `\\` × 3 = 3 個の backslash。escape 後は 6 個の backslash = `"\\\\\\\\\\\\"`。
- T4 の `\u{0301}` 等は Rust 文字列 literal で 16 進 Unicode escape。
- T5 の U+3000 全角空白は Rust 文字列 literal で直接 `'　'` も書ける。`\u{3000}` の方が visible。
- format string は uninlined_format_args 警告ゼロ維持のため `{result}` 形式で書く。
- `super::inline_markdown_text` は private fn だが既存 test (line 144 等) と同じく同モジュール内 `tests` から呼べる。
- 不要な `#[allow(...)]` は付けない。
