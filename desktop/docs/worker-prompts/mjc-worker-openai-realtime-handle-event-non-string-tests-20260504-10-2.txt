## 1. 背景 / Why
----------------------------------------------------------------------

`openai_realtime.rs` の private fn `ws_task::handle_event(text, pending, speaker, source)` (line 376-416) は、WebSocket から受け取った JSON を parse して以下の分岐を持つ:
1. JSON parse 失敗 → silent return
2. `type` field が non-string (null/number/array) または absent → silent return
3. `type == "conversation.item.input_audio_transcription.completed"` の場合:
   - `transcript` field が string で、かつ `.trim()` 後 non-empty なら `pending` に push
   - `transcript` が non-string、または trim 後 empty なら何もせず return
4. `type == "error"` の場合:
   - `error.message` (nested 1 階層) が string なら `push_error(...)` で error segment を push
   - `error.message` が non-string、または `error` 自体が message field を持たないなら何もせず return

既存 10 件 (line 476-751) は **正常パスと "type" 無関係 / 無効 JSON** をカバーしているが、**non-string field の silent return 分岐** と **trim 後 empty の skip 分岐** はカバーされていない。

これは前 mjc-main-20260504-9 のループ 3 で elevenlabs_realtime.rs の `is_scribe_error_event` boundary jets + `extract_error_message` の non-string fallback chain を 5 件で固定した作業 (`6760d58`) の **対称形**。openai 側は elevenlabs と違って fallback chain を持たず flat に「is_string でなければ silent return」する単純構造だが、その単純さこそ test で固定する価値がある (将来 elevenlabs の chain 構造に近づける誤改修や、逆に何かの fallback を追加してしまう foot-gun 防止)。

5 件で固定する契約:
- **silent return 契約**: `transcript` / `error.message` / `type` のいずれかが non-string でも `pending` 長は不変 (panic も abort もしない)
- **trim 空文字 skip 契約**: `transcript.trim()` が empty なら push せずに正常 return (whitespace-only transcript の SOX 漏れ防止)
- **error フィールド構造ロバスト性**: `error` field が `message` を持たない (`error: {}` や `error: null`) ケースで push_error されない

参考: 直近 SESSION SUMMARY (AGENT_LOG.md 末尾) の「次ループ候補 D'. openai_realtime.rs の対称補強 (規模 XS〜S、価値中、要事前 grep)」の実行版。grep 結果を踏まえて確定した 5 件案を以下に明示する。

## 2. 具体的 test 案 (assertion レベル)

### T1: handle_event_ignores_completed_event_when_transcript_field_is_non_string

**目的**: `conversation.item.input_audio_transcription.completed` 型 event で `transcript` field が non-string (null/number/array) の場合、silent return で `pending` 長 0 のまま (push されない) を固定。

```rust
#[test]
fn handle_event_ignores_completed_event_when_transcript_field_is_non_string() {
    let speaker = Some("自分".to_string());

    // ケース 1: transcript=null
    {
        let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
        ws_task::handle_event(
            r#"{"type":"conversation.item.input_audio_transcription.completed","transcript":null}"#,
            &pending,
            &speaker,
            Some(TranscriptionSource::Microphone),
        );
        assert_eq!(
            pending.lock().len(),
            0,
            "transcript=null は as_str() で None、silent return のはず"
        );
    }

    // ケース 2: transcript=number
    {
        let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
        ws_task::handle_event(
            r#"{"type":"conversation.item.input_audio_transcription.completed","transcript":42}"#,
            &pending,
            &speaker,
            Some(TranscriptionSource::Microphone),
        );
        assert_eq!(
            pending.lock().len(),
            0,
            "transcript=number は as_str() で None、silent return のはず"
        );
    }

    // ケース 3: transcript=array
    {
        let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
        ws_task::handle_event(
            r#"{"type":"conversation.item.input_audio_transcription.completed","transcript":["a","b"]}"#,
            &pending,
            &speaker,
            Some(TranscriptionSource::Microphone),
        );
        assert_eq!(
            pending.lock().len(),
            0,
            "transcript=array は as_str() で None、silent return のはず"
        );
    }
}
```

### T2: handle_event_ignores_completed_event_when_transcript_is_whitespace_only_or_empty

**目的**: `transcript` が string であっても `.trim()` 後 empty (" " / "\t" / "\n" / "" / 全角空白) の場合、push されないことを固定 (`!trimmed.is_empty()` 分岐保護)。

```rust
#[test]
fn handle_event_ignores_completed_event_when_transcript_is_whitespace_only_or_empty() {
    let speaker = Some("自分".to_string());
    // 4 種の空白系 + 全角空白 (U+3000) を一気に網羅。すべて trim 後 empty で push されない。
    for transcript in &["", "   ", "\t", "\n\n", "　　"] {
        let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
        let json = format!(
            r#"{{"type":"conversation.item.input_audio_transcription.completed","transcript":"{transcript}"}}"#
        );
        ws_task::handle_event(
            &json,
            &pending,
            &speaker,
            Some(TranscriptionSource::Microphone),
        );
        assert_eq!(
            pending.lock().len(),
            0,
            "trim 後 empty な transcript={transcript:?} では push されないはず"
        );
    }
}
```

## 3. 実装上の注意
----------------------------------------------------------------------

- **追加位置厳守**: `mod tests` (line 472〜) の **末尾** (`push_error_with_empty_and_multibyte_messages_preserve_text_format` の閉じ `}` のすぐ後、`mod tests` 自体の閉じ `}` の手前) に追加。既存 test の途中に挟まない。
- **既存 import で完結**: `use super::*;` (line 474) のみで全部参照可能。新たな `use` 行を足さない。
- **`r#"..."#` raw string で JSON literal を書く**: 既存 test と同じ書き方。embeddable な日本語 `自分` も raw string 内で OK。
- **T2 の format! 内の `{transcript}`**: `r#"{{...}}"#` 内では `{` `}` を `{{` `}}` でエスケープする既存パターンを踏襲。assertion メッセージの `{transcript:?}` は debug print で空白文字を可視化。
- **assertion メッセージは日本語**: 既存 test スタイル踏襲。
- **fmt 乱れ防止**: `cargo fmt` を最後に必ず通す。
- **重要な実装上の罠**:
  - line 386-389: `value.get("type").and_then(|v| v.as_str())` は **`type` field が absent でも non-string でも同じく `None`** を返す。両方の経路を T5 で網羅する。
  - line 408-413: `value.get("error").and_then(|e| e.get("message")).and_then(|m| m.as_str())` は **chain の各段で fallback が無い** (elevenlabs の `or_else` chain と対照的)。`error: "string"` のような flat case は `e.get("message")` で None になり、ここで fallback せず silent return が openai 側の現契約。これを T4 ケース 3 で固定。
  - line 396: `if !trimmed.is_empty()` の skip 分岐を T2 で固定。
