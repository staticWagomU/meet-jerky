### 追加する test 関数 3 件 (mod tests の末尾に追加)

**T1**: `whisper_request_params_debug_output_contains_struct_name_and_all_four_field_names_with_option_variants`
- 目的: WhisperRequestParams の Debug 出力に型名 + 全 4 snake_case field 名 + 値 + Option Some/None 両形態 + f32 が含まれる契約を CI 固定 (logger 経由のデバッグ性保護、特に `language` field の Some/None 区別表示と `temperature` の Float 表示形式を保護)。
- 手順:
  - `let with_lang = WhisperRequestParams { model: "small".to_string(), language: Some("ja".to_string()), response_format: "verbose_json".to_string(), temperature: 0.0 };`
  - `let without_lang = WhisperRequestParams { model: "large-v3".to_string(), language: None, response_format: "json".to_string(), temperature: 0.5 };`
  - `let with_dbg = format!("{with_lang:?}");`
  - `let without_dbg = format!("{without_lang:?}");`
  - assert: `with_dbg.contains("WhisperRequestParams")` (型名)
  - assert: `with_dbg.contains("model")`, `with_dbg.contains("language")`, `with_dbg.contains("response_format")`, `with_dbg.contains("temperature")` (4 snake_case field 名)
  - assert: `with_dbg.contains("\"small\"")`, `with_dbg.contains("\"ja\"")` (値、language は Some 中身として "ja" が出る)
  - assert: `with_dbg.contains("Some")` (Option Some variant 表示)
  - assert: `without_dbg.contains("None")` (Option None variant 表示)
  - assert: `without_dbg.contains("\"large-v3\"")`, `without_dbg.contains("\"json\"")`
  - assert: `with_dbg.contains("0.0")` (f32 0.0 の Debug 出力)
  - assert: `without_dbg.contains("0.5")` (f32 0.5 の Debug 出力)
  - 計 13 段程度の assert (1 型名 + 4 field 名 + 4 値 + Some/None + 2 f32)。

**T2**: `whisper_request_params_partial_eq_holds_reflexive_and_differs_per_field`
- 目的: WhisperRequestParams の `PartialEq` derive が field 単位独立に等値/不等値判定する契約を CI 固定 (`#[derive(PartialEq)]` の field 単位独立性を保護、特定 field を比較から除外する誤改修を遮断)。
- 手順:
  - `let base = WhisperRequestParams { model: "small".to_string(), language: Some("ja".to_string()), response_format: "verbose_json".to_string(), temperature: 0.0 };`
  - reflexive: `assert_eq!(base, base.clone());`
  - 4 fields 単位独立判定 (各 field を変えると不等):
    - `let m_diff = WhisperRequestParams { model: "large-v3".to_string(), ..base.clone() };` → `assert_ne!(base, m_diff);`
    - `let l_diff = WhisperRequestParams { language: Some("en".to_string()), ..base.clone() };` → `assert_ne!(base, l_diff);`
    - `let l_none = WhisperRequestParams { language: None, ..base.clone() };` → `assert_ne!(base, l_none);` (Option Some vs None も不等)
    - `let r_diff = WhisperRequestParams { response_format: "json".to_string(), ..base.clone() };` → `assert_ne!(base, r_diff);`
    - `let t_diff = WhisperRequestParams { temperature: 0.5, ..base.clone() };` → `assert_ne!(base, t_diff);`
  - 計 6 段の assert (1 reflexive + 5 不等)。

**T3**: `whisper_request_params_clone_is_deep_and_mutation_breaks_partial_eq`
- 目的: WhisperRequestParams の `Clone` derive が deep clone である契約を CI 固定。本 Loop は **PartialEq 派生型に対する Clone 独立性軸の新手法** = `assert_eq!(original, cloned)` で初期一致 → cloned 全 field を変更 → `assert_ne!(original, cloned)` + cloned の値を Debug 文字列で確認 + original の値を Debug 文字列で確認 (相互独立)。
- 手順:
  - `let original = WhisperRequestParams { model: "small".to_string(), language: Some("ja".to_string()), response_format: "verbose_json".to_string(), temperature: 0.0 };`
  - `let mut cloned = original.clone();`
  - `assert_eq!(original, cloned, "Clone 直後は等値");`
  - cloned 全 field mutate:
    - `cloned.model = "large-v3".to_string();`
    - `cloned.language = None;`
    - `cloned.response_format = "json".to_string();`
    - `cloned.temperature = 0.5;`
  - `assert_ne!(original, cloned, "全 field mutate 後は不等値");`
  - 念のため original 側の保持を Debug 文字列で確認 (PartialEq 直接判定との 2 重保険):
    - `let original_dbg = format!("{original:?}");`
    - `assert!(original_dbg.contains("\"small\""));`
    - `assert!(original_dbg.contains("Some"));`
    - `assert!(original_dbg.contains("\"ja\""));`
    - `assert!(original_dbg.contains("\"verbose_json\""));`
    - `assert!(original_dbg.contains("0.0"));`
    - `assert!(!original_dbg.contains("\"large-v3\""));`
    - `assert!(!original_dbg.contains("None"));`
    - `assert!(!original_dbg.contains("\"json\""));` ※注: `\"json\"` は full quote 付きで判定 (verbose_json には ":json" だが、独立 token としての "json" は別)
  - 注意: `\"json\"` の確認は微妙 = `\"verbose_json\"` の中に "json" が含まれるので `contains("\"json\"")` だと `\"verbose_json\"` には含まれない (ダブルクオート開始位置が違う) が、`contains("json")` だけだと両方マッチする。テストの意図 = original で `cloned.response_format = "json"` の変更が混入していないことを示すため、`!original_dbg.contains("\"json\",")` のようにカンマ込みで検証する形にすれば確実。または `!original_dbg.contains("response_format: \"json\"")` のように field 名 + 値で判定する。安全な書き方を選ぶこと。
  - assert!(!original_dbg.contains("0.5"));
  - 計 11 段程度の assert (1 初期等値 + 1 mutate 後不等 + 5 original 維持 contains + 4 cloned 値混入なし)。
