# 背景 / Why

`fn calculate_rms_from_sum(sum_squares: f32, sample_count: usize) -> f32` (L291-L301) は
マイク音声の声量計算 (UI レベルメーター駆動) の核心 fn です。

```rust
fn calculate_rms_from_sum(sum_squares: f32, sample_count: usize) -> f32 {
    if sample_count == 0 {
        return 0.0;
    }
    let rms = (sum_squares / sample_count as f32).sqrt();
    if rms.is_nan() {
        return 0.0;
    }
    rms.clamp(0.0, 1.0)
}
```

既存 3 件の直接テスト (count=0 / negative sum / clamp above) は重要 path をカバーしていますが、
以下の境界が **直接テスト未カバー**:

1. **`f32::NAN` sum_squares**: `is_nan()` の防御コードの存在意義
2. **`f32::INFINITY` sum_squares**: NaN → 0.0 と +Inf → 1.0 の対称性 (clamp の責務)
3. **中間値の sqrt 精度**: sum=4.0, count=16 → 0.5 (sqrt 演算の数値正確性)
4. **count=1 の単一サンプル境界**: sum=0.25, count=1 → 0.5
5. **clamp 上限ジャスト**: sum=1.0, count=1 → 1.0 (1.0 が減衰しない境界)

直接テスト追加の価値:
1. **`is_nan()` 防御コードの保護**: `if rms.is_nan() { return 0.0; }` を消すと
   NaN.clamp(0, 1) の挙動 (実装依存) でレベルメーターが NaN になり UI に化け文字が出る危険。
   T1 で消した瞬間に test が落ちる検知装置。
2. **NaN と +Inf の責務分離**: NaN → 0.0 (early return) / +Inf → 1.0 (clamp) で扱いが
   非対称。T1 と T2 で対称性が「意図的な非対称」であることを契約として明示。
3. **数値計算の正確性**: T3 で sqrt の中間値精度 (epsilon 1e-6) を保護。f32 演算精度の
   退行を検知。
4. **境界ジャスト**: T4/T5 で clamp 境界の上下 (count=1 で full-scale = 境界ジャスト) を固定。

# 具体的な実装変更

実装は変更しません。**`mod tests` 末尾**に以下の 5 件を追加してください。

各テストは独立した `#[test] fn` とすること。`use super::*;` は既存 (L582) で対応済み。

### T1. calculate_rms_from_sum_returns_zero_for_nan_sum_squares

**目的**: NaN sum_squares → 0.0 で `is_nan()` 防御コードの存在意義を保護。

```rust
#[test]
fn calculate_rms_from_sum_returns_zero_for_nan_sum_squares() {
    // NaN sum_squares は sqrt(NaN/count) = NaN を生み、is_nan() の早期 return で 0.0 になる。
    // is_nan() ブロックを消すと NaN.clamp(0, 1) の実装依存挙動で UI が化ける危険がある。
    // この防御コードの存在を契約として保護する。
    assert_eq!(calculate_rms_from_sum(f32::NAN, 4), 0.0);
}
```

### T2. calculate_rms_from_sum_clamps_positive_infinity_to_one

**目的**: +Inf sum_squares → 1.0 (clamp) で NaN との非対称な責務分離を明示。

```rust
#[test]
fn calculate_rms_from_sum_clamps_positive_infinity_to_one() {
    // +Inf.sqrt() は +Inf。is_nan() は false なので早期 return しない。
    // clamp(0, 1) で 1.0 に押さえられる。NaN → 0.0 と +Inf → 1.0 の非対称性は
    // 「異常値の種類別に意味のある最大/最小値を返す」設計意図の現れ。
    assert_eq!(calculate_rms_from_sum(f32::INFINITY, 4), 1.0);
}
```

### T3. calculate_rms_from_sum_returns_known_value_for_middle_range

**目的**: sqrt 演算の数値精度 (中間値) を eps 1e-6 で保護。

```rust
#[test]
fn calculate_rms_from_sum_returns_known_value_for_middle_range() {
    // sum=4.0, count=16 → sqrt(4.0 / 16.0) = sqrt(0.25) = 0.5
    // 中間値で sqrt 演算の数値精度退行を検知する。
    // f32 sqrt の精度は IEEE 754 レベルで保証されるが、誤って sqrt を消すリファクタや
    // sample_count を i32 に変えると精度が落ちる可能性を検知する。
    assert_close(calculate_rms_from_sum(4.0, 16), 0.5, 1e-6);
}
```

### T4. calculate_rms_from_sum_handles_single_sample_count

**目的**: count=1 の単一サンプル境界で sqrt(sum) = 期待値であること。

```rust
#[test]
fn calculate_rms_from_sum_handles_single_sample_count() {
    // count=1 では sum / count = sum なので sqrt(sum) が直接 RMS になる。
    // sum=0.25 → sqrt(0.25) = 0.5。count=1 の境界で 0 除算や u32 cast の罠を検知する。
    assert_close(calculate_rms_from_sum(0.25, 1), 0.5, 1e-6);
}
```

### T5. calculate_rms_from_sum_at_clamp_upper_boundary

**目的**: clamp 上限ジャスト (1.0) が減衰しないことを境界として保護。

```rust
#[test]
fn calculate_rms_from_sum_at_clamp_upper_boundary() {
    // sum=1.0, count=1 → sqrt(1.0) = 1.0。clamp(0, 1) は 1.0 を 1.0 のまま通す。
    // 境界ジャスト値で「過剰な丸め」や「< 1.0 への押し下げ」リファクタを検知する。
    assert_close(calculate_rms_from_sum(1.0, 1), 1.0, 1e-6);
}
```
