## T1: segment_to_append_args_falls_back_to_stream_started_when_zero_start_and_observed_none
- 目的: zero_start segment (start_ms=0, end_ms=0) を observed=None で渡した時に `observed_at_secs.unwrap_or(stream_started_at_secs)` の None 分岐 (= fallback to stream_started_at_secs) が happy path で動作することを CI 固定。
- 既存 test (`zero_start_segment_uses_observed_time_when_available`) は observed=Some(1075) のみ、`offset_saturates_to_zero_when_segment_time_precedes_session_start` は has_engine_timestamp=true 経路 (start_ms=0, end_ms=100) のみ。**zero_start + observed=None + happy 経路** は未保護。
- Setup:
  - segment.start_ms = 0, end_ms = 0, speaker = Some("自分"), text = "test"
  - session_started_at_secs = 1_000, stream_started_at_secs = 1_040
  - segment_to_append_args (= observed=None 内部固定) を呼ぶ
- Assert: offset == 40 (= stream_started 1040 - session_started 1000)
- コメント: 「zero_start segment + observed=None で `unwrap_or(stream_started_at_secs)` の None 分岐が動く契約。observed=None 経路は segment_to_append_args の内部固定。」

## T2: segment_to_append_args_trims_unicode_full_width_space_in_text
- 目的: `segment.text.trim().to_string()` (line 45) の U+3000 (全角空白) trim 契約を CI 固定。既存 test は ASCII whitespace ("hello   world") のみ。`normalize_speaker_trims_unicode_full_width_space_u3000` (T1 系既存 test) は speaker 側だけ網羅、text 側は未保護。
- Setup:
  - segment.text = "\u{3000}\u{3000}hello\u{3000}\u{3000}", start_ms=2000, end_ms=3500, speaker=Some("自分")
  - segment_to_append_args(&segment, 1000, 1040) を呼ぶ
- Assert: text == "hello" (前後の U+3000 が trim される。内部 "hello" はそのまま)
- コメント: 「U+3000 は str::trim 対象 (Unicode White_Space プロパティ準拠)。speaker 側で確認済の現契約を text 側でも固定。」

## T3: segment_to_append_args_returns_empty_text_when_text_is_only_whitespace
- 目的: text が全て whitespace (ASCII + Unicode 混在) の場合に `text.trim().to_string()` が空文字列 `""` を返す契約を CI 固定。「empty 時に "(empty)" 等のフォールバックを後付け」する誤改修への検知装置。
- Setup:
  - segment.text = "\u{3000}\t\n  \u{3000}", start_ms=2000, end_ms=3500, speaker=Some("自分")
  - segment_to_append_args(&segment, 1000, 1040) を呼ぶ
- Assert: text == "" (空文字列)
- コメント: 「text 全 whitespace → trim 後 "" を返す契約。empty 時の fallback (例: "(空)" 等) を後付けする誤改修への検知。speaker 側 (`_is_trimmed_and_none_falls_back_to_unknown_label` の None fallback) と非対称な契約 = text には "不明" 相当のフォールバックは存在しない。」
