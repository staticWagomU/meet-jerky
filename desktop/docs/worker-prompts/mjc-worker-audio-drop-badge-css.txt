# 背景
直前 Loop 1 (commit 353da45) で `audio-drop-count` UI 表示を集約 1 ブロックから per-source badge に移行した際、新規 CSS class 追加を回避するため className `audio-source-silence-badge` を流用した。これにより drop badge は「入力待ち」と同じグレー pill 表示になっており、視覚的に「警告」と認識しにくい。

本ループはその意図的な妥協を解消する Tidy After 的 polish:
- 新規 className `audio-source-drop-badge` を追加 (silence-badge と同じサイズ・形状 + meeting-alert と同じ警告色)
- MicrophoneSection / SystemAudioSection の drop badge の className を新規 class に切り替え

# Why
1. **意味と視覚の一致**: silence-badge (グレー = neutral) と drop-badge (赤系 = warning) で意味の異なる health 指標を視覚的に分離
2. **既存パターンとの完全対称**: silence-badge のサイズ・形状 + meeting-alert の警告色 = 既存デザイン語彙の組み合わせのみで新規デザイン要素を導入しない
3. **将来の拡張余地**: drop count threshold 別の色階調 (yellow → red) など、独自進化させる際の単独 class が確保される
