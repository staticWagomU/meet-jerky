# 背景
audio-drop-count chain (rust → Tauri event → frontend listener → 累積 state → UI 表示 → session start reset) は前セッション群 (mjc-main-13/14/15) で一気通貫実装済みです。現在の UI 表示は src/routes/TranscriptView.tsx の line 2370-2381 で「マイク {N} サンプル / システム音声 {N} サンプル」を 1 ブロックに合算して表示しています。これは「メニューバー上部のサマリ」位置にあり、各音声 source の文脈 (MicrophoneSection / SystemAudioSection) からは物理的に離れています。

product-concept 優先度 9「録音状態の透明性」のさらなる強化として、各音声 source の文脈 (MicrophoneSection / SystemAudioSection の header) 内に per-source の drop count badge を配置し、UI コンテキストとの一致を高めます (前 mjc-main-15 SESSION SUMMARY の最有力候補 A4 = per-source UI badge prop drill)。

# Why (この変更の価値 3 つ)
1. **コンテキスト一致**: drop count は本来「該当の音声 source の問題指標」であり、その source の状態 badge (録音中/未録音/入力待ち) と並ぶことで認知負荷が下がる。
2. **集約の重複表示解消**: 集約サマリ (line 2370-2381) を per-source 表示に置き換えることで「mic と system 合算の意味の分かりにくさ」を解消する。
3. **既存 silence-badge との対称構造**: MicrophoneSection / SystemAudioSection の header にすでに `audio-source-silence-badge` (入力待ち) という同種の health 指標 badge があり、その隣に並ぶ形で完全対称な構造を取れる (= worker は既存パターンを完全コピーするだけで済む)。

# 実装上の注意
- prop を **必須 (`?` なし) にする** ことで、TranscriptView 側の prop pass 漏れがあれば tsc 型エラーで弾かれる = 検知装置として機能する。
- 既存 className `audio-source-silence-badge` を流用する判断 = 新規 CSS class を増やさず、既存のグレー背景・小型 badge スタイルがそのまま使える。色やサイズの微調整 (例: drop は赤系) は次ループの polish に分離。今回はとにかく「per-source への移行を 1 commit で完結」が目的。
- isCompact / isMeetingActive 等の既存 prop は **触らない**。
- audio-drop-count event listener / payload validation / reset semantics は **触らない**。
- 表示テキストは「drop {N}」「破棄 {N}」のどちらか。日本語で簡潔に「破棄 {N}」が product-concept 優先度 9 のメッセージ性に合う (ユーザーが「これは破棄の意味」と即時理解可能) → 推奨は「破棄 {N}」。判断は worker に委ねる。
- noUnusedLocals / noUnusedParameters は tsconfig で true なので、prop を destructure しただけで使っていないと型エラーになる → JSX で実際に参照すること。
- 削除対象の line 2370-2381 ブロックはちょうど 12 行 (空行除く)。前後の `audioDropCountListenerError` block (2360-2368) と `lastSavedPath` block (2382-) は維持。
