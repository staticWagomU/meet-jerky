  fn normalize_speaker(raw: Option<&str>) -> String {
      match raw.map(str::trim) {

      }
  }

## 追加するテスト 5 件 (関数名はそのまま使う)

### A. `normalize_speaker_returns_unknown_for_none`
- 振る舞い: `normalize_speaker(None) == "不明"`

### B. `normalize_speaker_returns_unknown_for_empty_string`
- 振る舞い: `normalize_speaker(Some("")) == "不明"`

### C. `normalize_speaker_returns_unknown_for_whitespace_only`
- 振る舞い: `normalize_speaker(Some("   ")) == "不明"` および `normalize_speaker(Some("\t\n  ")) == "不明"` の 2 assertion (whitespace 文字種を 2 種類確認)

### D. `normalize_speaker_passes_through_normal_label`
- 振る舞い: `normalize_speaker(Some("Alice")) == "Alice"` および `normalize_speaker(Some("自分")) == "自分"` および `normalize_speaker(Some("相手側")) == "相手側"` の 3 assertion (英数字 + 日本語 + プロジェクト固有ラベル)

### E. `normalize_speaker_trims_surrounding_whitespace_only`
- 振る舞い:
  - `normalize_speaker(Some("  Alice  ")) == "Alice"` (前後空白 trim、内部に空白なし)
  - `normalize_speaker(Some(" Alice Bob ")) == "Alice Bob"` (前後 trim、**内部の空白は保持**) ← この 2 件目が重要 (内部 collapse しない契約)

## 設計意図 (なぜこれをやるか)
- 既存 `speaker_is_trimmed_and_none_falls_back_to_unknown_label` は `segment_to_append_args` 経由の間接テストのみ。`normalize_speaker` を別関数に抽出/インライン化するリファクタを行うと「責任は `normalize_speaker` か `segment_to_append_args` か」の切り分けができない。
- 直接呼び出しテストで `normalize_speaker` の全分岐 (None / 空文字列 / whitespace-only / 通常 / trim 必要) を裏付けることで、リファクタ耐性を獲得し、また「内部空白を collapse しない」契約を明示する。
- `Some(" Alice Bob ")` → `"Alice Bob"` のテストは特に重要: trim は前後のみ、内部空白は保持する契約 (会議で複数人の話者ラベル "Alice Bob" のような連名対応の余地を残すため)。
- 前セッション (audio_utils, session_store) で確立した「private fn 直接テスト」パターンの正統な拡張。リスク極小。
