# 背景 / Why

`mod ws_task` 内の `fn push_error` (L418-L432) は WebSocket エラーイベントから
TranscriptionSegment を生成する核心 fn ですが、現状は `handle_event` 経由の
**間接テストのみ** (`error_events_are_queued_as_error_segments` L506,
`handle_event_pushes_error_message` L562) でカバーされています。

直接テストを追加する価値:
1. **prefix 文言契約の固定**: `text = format!("[OpenAI Realtime エラー: {message}]")`
   の prefix `"[OpenAI Realtime エラー: "` と suffix `"]"` をユーザー向け文言として固定。
   将来 prefix が削除/変更されたら CI が落ちて気付ける。
2. **不変条件の固定**: `start_ms=0, end_ms=0, is_error=Some(true)` の 3 値を 1 関数で
   契約強制。エラーセグメントが時刻情報を持たない・UI 側 error 装飾駆動という設計を保護。
3. **source/speaker passthrough 契約**: `Some(TranscriptionSource::Microphone)` / `None`
   の双方向 passthrough と speaker の clone 動作を直接固定。
4. **関数分離リファクタへの耐性**: 将来 push_error がインライン化/別 fn 統合された場合、
   間接テストでは fn 自体の存在検知ができない。直接テストで fn 単体の契約を保護。
5. **elevenlabs_realtime との対称性**: elevenlabs 側は既に同種 private fn 3 つを直接
   テスト済 (commit 8235878)。openai 側もテストすることで、両 Realtime engine の
   error path 不変条件を同水準で保護できる。

### T1. push_error_format_prefix_and_suffix_are_fixed

**目的**: ユーザー向け prefix `"[OpenAI Realtime エラー: "` と suffix `"]"` の完全一致契約。

```rust
#[test]
fn push_error_format_prefix_and_suffix_are_fixed() {
    let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
    let speaker = Some("自分".to_string());

    ws_task::push_error(
        &pending,
        &speaker,
        Some(TranscriptionSource::Microphone),
        "connection closed".to_string(),
    );

    let segments = pending.lock();
    assert_eq!(segments.len(), 1);
    // prefix と suffix を含む完全一致で文言契約を強制。
    // prefix を削る・bracket を消すリファクタを CI で即座検知する。
    assert_eq!(
        segments[0].text,
        "[OpenAI Realtime エラー: connection closed]"
    );
}
```

### T2. push_error_sets_zero_timestamps_and_is_error_true

**目的**: エラーセグメントの time-less + is_error=Some(true) 不変条件を 1 関数で固定。

```rust
#[test]
fn push_error_sets_zero_timestamps_and_is_error_true() {
    let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
    let speaker = Some("自分".to_string());

    ws_task::push_error(
        &pending,
        &speaker,
        Some(TranscriptionSource::Microphone),
        "rate limit".to_string(),
    );

    let segments = pending.lock();
    assert_eq!(segments.len(), 1);
    // エラーセグメントは時刻情報を持たない・is_error=Some(true) で UI 側 error 装飾を駆動。
    // この 3 値同時が「エラーセグメント」の identity 契約。
    assert_eq!(segments[0].start_ms, 0);
    assert_eq!(segments[0].end_ms, 0);
    assert_eq!(segments[0].is_error, Some(true));
}
```

### T3. push_error_passes_source_through

**目的**: source の Option<TranscriptionSource> passthrough 契約 (Some / None 両方向)。

```rust
#[test]
fn push_error_passes_source_through() {
    // ケース 1: Some(Microphone) は Some(Microphone) のまま渡る
    {
        let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
        let speaker = Some("自分".to_string());
        ws_task::push_error(
            &pending,
            &speaker,
            Some(TranscriptionSource::Microphone),
            "err".to_string(),
        );
        assert_eq!(pending.lock()[0].source, Some(TranscriptionSource::Microphone));
    }

    // ケース 2: None は None のまま渡る
    {
        let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
        let speaker = Some("自分".to_string());
        ws_task::push_error(&pending, &speaker, None, "err".to_string());
        assert_eq!(pending.lock()[0].source, None);
    }
}
```

### T4. push_error_clones_speaker_field

**目的**: speaker の Option<String> clone 動作 (Some / None 両方向)。

```rust
#[test]
fn push_error_clones_speaker_field() {
    // ケース 1: Some("自分") が clone されて渡る (元の所有権は保持される)
    {
        let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
        let speaker = Some("自分".to_string());
        ws_task::push_error(
            &pending,
            &speaker,
            Some(TranscriptionSource::Microphone),
            "err".to_string(),
        );
        assert_eq!(pending.lock()[0].speaker, Some("自分".to_string()));
        // 元の speaker が clone 後も有効であることを確認 (move されていない)
        assert_eq!(speaker, Some("自分".to_string()));
    }

    // ケース 2: None は None のまま渡る
    {
        let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
        let speaker: Option<String> = None;
        ws_task::push_error(
            &pending,
            &speaker,
            Some(TranscriptionSource::Microphone),
            "err".to_string(),
        );
        assert_eq!(pending.lock()[0].speaker, None);
    }
}
```

### T5. push_error_with_empty_message_still_produces_brackets

**目的**: 空 message でも prefix/suffix を保つ現挙動の固定 (空 message でも UI 上はエラー
セグメントとして識別可能)。multibyte message 含む現状の text 埋め込み挙動も併せて保護。

```rust
#[test]
fn push_error_with_empty_and_multibyte_messages_preserve_text_format() {
    // ケース 1: 空 message でも prefix/suffix が残り "[OpenAI Realtime エラー: ]" になる
    {
        let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
        let speaker = Some("自分".to_string());
        ws_task::push_error(
            &pending,
            &speaker,
            Some(TranscriptionSource::Microphone),
            "".to_string(),
        );
        // 空 message でも UI 側でエラー判定できるよう bracket だけは残す現契約。
        assert_eq!(pending.lock()[0].text, "[OpenAI Realtime エラー: ]");
    }

    // ケース 2: 日本語マルチバイト message が prefix/suffix の間にそのまま埋め込まれる
    {
        let pending = Arc::new(Mutex::new(Vec::<TranscriptionSegment>::new()));
        let speaker = Some("自分".to_string());
        ws_task::push_error(
            &pending,
            &speaker,
            Some(TranscriptionSource::Microphone),
            "認証エラーが発生しました".to_string(),
        );
        // multibyte の余分な escape/sanitize がないことを契約として固定。
        assert_eq!(
            pending.lock()[0].text,
            "[OpenAI Realtime エラー: 認証エラーが発生しました]"
        );
    }
}
```
