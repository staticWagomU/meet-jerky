### 追加する test 関数 3 件 (mod tests の末尾に追加)

**T1**: `app_settings_debug_output_contains_struct_name_and_all_six_field_names`
- 目的: AppSettings の Debug 出力が型名 + 全 6 snake_case field 名 + 値を含むことを CI 固定 (Frontend 側のログ可読性 / 内部 logger 経由でのデバッグ性を保護)。
- 手順:
  - `let settings = AppSettings { transcription_engine: TranscriptionEngineType::AppleSpeech, whisper_model: "base".to_string(), microphone_device_id: Some("dev-mic-001".to_string()), language: "ja".to_string(), output_directory: Some("/tmp/out".to_string()), api_key: Some("sk-test-001".to_string()) };`
  - `let debug = format!("{settings:?}");`
  - assert: `"AppSettings"` を含む (型名)
  - assert: `"transcription_engine"`, `"whisper_model"`, `"microphone_device_id"`, `"language"`, `"output_directory"`, `"api_key"` を全て含む (6 つの snake_case field 名)
  - assert: `"AppleSpeech"`, `"base"`, `"dev-mic-001"`, `"ja"`, `"/tmp/out"`, `"sk-test-001"` を全て含む (6 つの値)
  - 計 13 段の `assert!(debug.contains(...))` 単位 assert。

**T2**: `app_settings_clone_is_deep_and_does_not_mutate_original`
- 目的: AppSettings の `Clone` derive が deep clone である契約を CI 固定 (Rc<T> shallow 化 / Cow 化 誤改修を遮断)。
- 手順:
  - `let original = AppSettings { transcription_engine: TranscriptionEngineType::Whisper, whisper_model: "small".to_string(), microphone_device_id: Some("mic-A".to_string()), language: "auto".to_string(), output_directory: None, api_key: Some("k1".to_string()) };`
  - `let mut cloned = original.clone();`
  - cloned のすべての field を別の値に変更:
    - `cloned.transcription_engine = TranscriptionEngineType::OpenAIRealtime;`
    - `cloned.whisper_model = "large-v3".to_string();`
    - `cloned.microphone_device_id = Some("mic-B".to_string());`
    - `cloned.language = "en".to_string();`
    - `cloned.output_directory = Some("/var/log".to_string());`
    - `cloned.api_key = None;`
  - assert (Debug 出力で original が変わっていないことを確認、PartialEq がないため Debug 文字列で間接的に判定):
    - `let original_debug = format!("{original:?}");`
    - `assert!(original_debug.contains("Whisper"));`
    - `assert!(!original_debug.contains("OpenAIRealtime"));`
    - `assert!(original_debug.contains("\"small\""));`
    - `assert!(!original_debug.contains("\"large-v3\""));`
    - `assert!(original_debug.contains("\"mic-A\""));`
    - `assert!(!original_debug.contains("\"mic-B\""));`
    - `assert!(original_debug.contains("\"auto\""));`
    - `assert!(!original_debug.contains("\"en\""));`
    - `assert!(original_debug.contains("None"));` (output_directory が None のまま)
    - `assert!(!original_debug.contains("\"/var/log\""));`
    - `assert!(original_debug.contains("\"k1\""));` (api_key)
  - 計 11 段の assert で Clone 独立性を field 単位独立に確認。

**T3**: `app_settings_serde_serialize_uses_camel_case_for_all_six_fields`
- 目的: AppSettings の serde camelCase JSON key 名固定 (TypeScript Frontend / settings.json への書き出し contract を CI 保護、`#[serde(rename_all = "camelCase", default)]` 誤削除を遮断)。
- 手順:
  - `let settings = AppSettings { transcription_engine: TranscriptionEngineType::ElevenLabsRealtime, whisper_model: "tiny".to_string(), microphone_device_id: Some("device-X".to_string()), language: "ja".to_string(), output_directory: Some("/home/u/out".to_string()), api_key: Some("eleven-key".to_string()) };`
  - `let json = serde_json::to_value(&settings).expect("serialize ok");`
  - `let obj = json.as_object().expect("object");`
  - assert: `obj.len() == 6` (field 数厳密)
  - assert: 6 つの camelCase key が存在 (`transcriptionEngine`, `whisperModel`, `microphoneDeviceId`, `language`, `outputDirectory`, `apiKey`)
  - assert: 6 つの snake_case key が **存在しない** (`transcription_engine`, `whisper_model`, `microphone_device_id`, `output_directory`, `api_key` … `language` は元から snake_case 形態と区別なしなので除外)
  - assert: 値の正しさ (`obj["transcriptionEngine"] == "elevenLabsRealtime"` (TranscriptionEngineType も serde camelCase なので nested で `elevenLabsRealtime` に serialize される、これは mjc-main-34 Loop 2 で固定済み), `obj["whisperModel"] == "tiny"`, `obj["microphoneDeviceId"] == "device-X"`, `obj["language"] == "ja"`, `obj["outputDirectory"] == "/home/u/out"`, `obj["apiKey"] == "eleven-key"`)
  - 計 17 段程度の assert (1 len + 6 contains_key + 5 !contains_key + 6 値正しさ)。
