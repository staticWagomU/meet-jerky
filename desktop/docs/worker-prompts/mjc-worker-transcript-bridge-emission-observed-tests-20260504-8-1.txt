## 1. 背景 / Why
`build_append_args_for_emission_at` (line 74-90) は live transcription emission の核となる純粋関数で、以下 2 段階の早期リターンを持つ:

```rust
pub fn build_append_args_for_emission_at(
    segment: &TranscriptionSegment,
    session_started_at_secs: Option<u64>,
    stream_started_at_secs: u64,
    observed_at_secs: Option<u64>,
) -> Option<(String, u64, String)> {
    if segment.is_error.unwrap_or(false) {
        return None;
    }
    let started = session_started_at_secs?;
    Some(segment_to_append_args_at(segment, started, stream_started_at_secs, observed_at_secs))
}
```

既存テスト 16 件のうち `build_append_args_for_emission` (test-only wrapper、observed_at_secs=None 固定、line 58-69) 経由のものは 4 件:
- `build_append_args_saturates_offset_when_stream_precedes_session` (Some, observed=None)
- `build_append_args_returns_some_with_same_values_as_segment_to_append_args` (Some, observed=None)
- `build_append_args_returns_none_when_session_not_started` (None, observed=None)
- `build_append_args_returns_none_for_error_segments` (Some, observed=None)

**Gap**: `build_append_args_for_emission_at` を **直接呼び出し**、`observed_at_secs = Some(...)` を渡す経路がゼロ。
- error 短絡が observed 経路でも有効か
- session 未開始の早期リターンが observed があっても優先か
- zero_start segment + observed=Some の挙動が `segment_to_append_args_at` 経由で正しく接続されているか
- engine timestamp ありなら observed が無視される挙動が emission helper 経由でも維持されるか
- 「error 短絡 → session_started? の順序」の優先順位 (first-violation contract)

これらを 5 件の direct test で固定する。

## 2. 具体的 test 案 (assertion レベル)

`super::*` を use している tests mod 内なので `build_append_args_for_emission_at` を直接呼べる。ヘルパー `sample_segment()` は既存にある (line 109-118)。

### T1. `build_append_args_for_emission_at_returns_none_for_error_segment_even_with_observed_time`
```rust
let mut segment = sample_segment();
segment.is_error = Some(true);
let result = build_append_args_for_emission_at(&segment, Some(1_000), 1_000, Some(1_075));
assert!(result.is_none(), "is_error の早期リターンは observed_at_secs があっても有効");
```
**意図**: error 短絡が observed 経路でも変わらず None を返す現契約を固定。

### T2. `build_append_args_for_emission_at_returns_none_when_session_not_started_with_observed_time`
```rust
let segment = sample_segment(); // is_error = None
let result = build_append_args_for_emission_at(&segment, None, 1_000, Some(1_075));
assert!(result.is_none(), "session_started_at_secs == None なら observed があっても None");
```
**意図**: session 未開始の早期リターンが observed があっても優先される現契約を固定。

### T3. `build_append_args_for_emission_at_uses_observed_time_for_zero_start_segment`
```rust
// Realtime 系の zero_start segment (start_ms=0, end_ms=0) は observed_at_secs に倒れる
let segment = TranscriptionSegment {
    text: "realtime".to_string(),
    start_ms: 0,
    end_ms: 0,
    source: None,
    speaker: Some("相手側".to_string()),
    is_error: None,
};
let result = build_append_args_for_emission_at(&segment, Some(1_000), 1_000, Some(1_075))
    .expect("observed があれば Some を返す");
assert_eq!(result.0, "相手側");
assert_eq!(result.1, 75, "zero_start segment は observed_at_secs - session_started_at_secs を使う");
assert_eq!(result.2, "realtime");
```
**意図**: emission helper 経由でも `segment_to_append_args_at` の zero_start ロジックが正しく接続されていることを保証。

### T4. `build_append_args_for_emission_at_keeps_engine_timestamp_when_present_ignoring_observed_time`
```rust
// engine timestamp ありなら observed は無視される
let segment = sample_segment(); // start_ms=2000, end_ms=3500, speaker="自分", text="こんにちは"
let result = build_append_args_for_emission_at(&segment, Some(1_000), 1_040, Some(9_999))
    .expect("Some を返す");
assert_eq!(result.0, "自分");
assert_eq!(result.1, 42, "engine timestamp あり時は observed を無視し engine timestamp が勝つ");
assert_eq!(result.2, "こんにちは");
```
**意図**: emission helper 経由でも「engine timestamp 優先」の現契約が維持されていることを保証。observed=9_999 という極端値を渡しても 42 になることで「engine timestamp が確実に勝つ」を強く示す。

### T5. `build_append_args_for_emission_at_short_circuits_on_error_before_checking_session_started`
```rust
// is_error=true かつ session_started_at_secs=None。
// 実装は is_error チェックが先なので、segment_started がなくても is_error の None を返す経路を通る。
// (もし順序が逆になった場合、segment_started?.None で先に None になり、`?` で None を返すが、
//  実装上の優先順位を first-violation contract として明示的に固定する。)
let mut segment = sample_segment();
segment.is_error = Some(true);
let result = build_append_args_for_emission_at(&segment, None, 1_000, Some(1_075));
assert!(result.is_none(), "is_error と session 未開始の二重違反でも None (どちらが先でも結果同じ)");
```
**意図**: 二重違反の現契約 (どちらでも None) を固定。is_error と session check の順序は実装詳細だが、結果が None である契約を保証。

## 3. 実装上の注意

- `super::*` は既に use 済み。`TranscriptionSegment` は use 不要 (super 経由で見える)。
- `sample_segment()` は line 109-118 にあるヘルパーで、既存テストでも使われている。**そのまま使うこと**。
- T3 / T4 の数値の意味:
  - T3: stream=1000 + observed=1075 → segment_abs_secs = 1075 → offset = 1075 - 1000 = 75
  - T4: stream=1040 + start_ms=2000 → segment_offset_from_stream_secs = 2 → segment_abs_secs = 1042 → offset = 1042 - 1000 = 42
  - これらは line 280-298 の `happy_path_forwards_text_and_computes_offset` と同じ計算ルールで、既存テストとの整合性を確認するための値。
- **既存テストとの重複回避**: T1 と既存 `build_append_args_returns_none_for_error_segments` (line 268-277) は似ているが、後者は `build_append_args_for_emission` (observed=None 固定) 経由。T1 は `_at` を直接呼んで observed=Some(1075) を渡す違いを assertion comment で明示。
- **T2 と既存 `build_append_args_returns_none_when_session_not_started`** (line 259-265) も同様の違い。コメントで観点を明示すること。
- **コメントは日本語、簡潔に** (AGENTS.md 準拠)。
- **eprintln や panic 系の assert はしない** (純粋関数なので不要)。
