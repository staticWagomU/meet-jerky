## 1. 新関数 `is_goto_legacy_meeting_url` を `app_detection.rs` に追加

L944 の `is_goto_meeting_url` の直後 (L945 周辺) に追加:

```rust
fn is_goto_legacy_meeting_url(host: &str, path: &str) -> bool {
    if host != "global.gotomeeting.com" {
        return false;
    }
    let Some(id) = path.strip_prefix("/join/") else {
        return false;
    };
    let id = id.strip_suffix('/').unwrap_or(id);
    id.len() == 9 && id.chars().all(|c| c.is_ascii_digit())
}
```

**設計判断**:
- host は `global.gotomeeting.com` 完全一致のみ (subdomain 対応なし) = legacy 形式は subdomain を持たない
- path は `/join/<9-digit-numeric>` のみ = 9 桁 numeric ID は GoToMeeting の最も documented standard
- 末尾 `/` (trailing slash) は許容 (`strip_suffix('/')`)
- ID が他の長さや非数字を含む場合は false (DNS spoofing と同等の URL spoofing 防御)
- ファイル内既存 `is_goto_meeting_url` と完全に独立 = path validation logic が異なる (room 名 vs 9-digit-id)
