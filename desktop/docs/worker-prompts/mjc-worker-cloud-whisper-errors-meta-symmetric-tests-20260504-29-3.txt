## 背景・目的

`cloud_whisper_errors.rs` (361 行) の既存 19 test は status 軸 (401/429/5xx/400/499/600/200/u16 端点/隣接値) と body 軸 (空/multibyte/200 char 境界/whitespace normalize/multiline+control chars) と漏洩防止軸 (401/429/500/599 で body 無視) と pure function 軸 (idempotency / status 数値が message に漏れない / 異 status 同 body で message 一致) を網羅。

handoff サマリの「対称軸補強」パターン (型/経路/フィールド/層 の 4 特殊形) を踏まえ、本 Loop は「**メタ対称軸補強**」= status 軸 / body 軸 / pure function 軸の **異なる対称軸を 1 ループで一通り application** する 3 軸構造を採用。

未保護軸:
1. **status 軸 (3xx redirect 範囲)**: 既存 test は 200 / 400-499 / 500-599 / 600 / 0 / 1 / u16::MAX をカバーするが、3xx 範囲 (301/302/304 等の redirect) は未保護
2. **body 軸 (NUL byte の passthrough)**: 既存 test は whitespace 系 (`\n` `\t` `\r` `空白`) を split_whitespace で除去するが、NUL byte (`\0`) は `char::is_whitespace()` で false なので passthrough されるはず — この性質は test 0
3. **pure function 軸 (同 status × 異 body の message 差異)**: 既存 T19 (l.319 `classify_other_is_pure_function_with_status_in_field_only_not_in_message`) は idempotency (同 status × 同 body) と「異 status × 同 body で message 一致」をカバーするが、対称軸の片方「同 status × 異 body で message 差異」は未保護

## 追加する test 3 件 (命名と意図を厳守)

### T1: `classify_returns_other_for_3xx_redirect_status_range_without_falling_through_to_5xx`

- 目的: 3xx (redirect) status (301 / 302 / 304) は 500..=599 範囲外なので Other に落ちる現契約を CI 固定。「将来 3xx を別 variant にする誤改修」「3xx を ServerError に誤分類する誤改修」を遮断。
- 実装:
  ```rust
  for (status, body) in [
      (301, "moved permanently"),
      (302, "found"),
      (304, "not modified"),
  ] {
      assert_eq!(
          classify_cloud_whisper_error(status, body),
          CloudWhisperError::Other {
              status,
              message: body.to_string(),
          },
          "{status} (3xx redirect) は 500..=599 範囲外で Other に落ちる契約 (ServerError ではない)"
      );
  }
  ```

### T2: `classify_other_preserves_nul_byte_in_body_message_because_nul_is_not_whitespace`

- 目的: NUL byte (`\0`) は `char::is_whitespace()` で false なので `split_whitespace` で除去されず、message に passthrough する現契約を CI 固定。「将来 NUL byte を sanitize で消す誤改修」「NUL を whitespace 扱いする誤改修」を遮断。
- 実装:
  ```rust
  let body = "abc\0def";
  let CloudWhisperError::Other { message, .. } = classify_cloud_whisper_error(400, body) else {
      panic!("400 should classify as Other");
  };
  assert_eq!(
      message, "abc\0def",
      "NUL byte は char::is_whitespace で false なので split_whitespace で除去されず passthrough する契約"
  );
  assert!(
      message.contains('\0'),
      "message に NUL byte が含まれる (sanitize で消されない)"
  );
  ```

### T3: `classify_other_with_same_status_yields_distinct_messages_for_distinct_bodies_pure_function_inverse_axis`

- 目的: 同 status × 異 body で message が異なる現契約を CI 固定。既存 T19 (異 status × 同 body で message 一致) の「逆対称軸」(pure function の入力空間の独立性) を補強。「将来 message を status 依存にする誤改修」と相補的に「message を body 非依存にする誤改修」を遮断。
- 実装:
  ```rust
  let a = classify_cloud_whisper_error(404, "body alpha");
  let b = classify_cloud_whisper_error(404, "body beta");
  if let (
      CloudWhisperError::Other { status: status_a, message: msg_a },
      CloudWhisperError::Other { status: status_b, message: msg_b },
  ) = (a, b)
  {
      assert_eq!(status_a, 404);
      assert_eq!(status_b, 404);
      assert_eq!(msg_a, "body alpha");
      assert_eq!(msg_b, "body beta");
      assert_ne!(
          msg_a, msg_b,
          "同 status (404) × 異 body で message は異なる契約 (pure function: body は message に passthrough、status の値だけで上書きしない)"
      );
  } else {
      panic!("both 404 calls should classify as Other");
  }
  ```
