## T1: build_append_args_for_emission_at_falls_back_to_stream_started_when_zero_start_and_observed_none
- 目的: emission helper 経由 (`build_append_args_for_emission_at`) で zero_start segment (start_ms=0, end_ms=0) + observed=None を渡した時に segment_to_append_args_at 内部の `observed_at_secs.unwrap_or(stream_started_at_secs)` フォールバックが動作する契約を CI 固定。
- 既存 test (`build_append_args_for_emission_at_uses_observed_time_for_zero_start_segment`) は observed=Some(1075) のみ、`build_append_args_returns_some_with_same_values_as_segment_to_append_args` は engine timestamp 経路のみ。**emission helper 経由の zero_start + observed=None** 経路は未保護。
- Setup:
  - segment.start_ms = 0, end_ms = 0, speaker = Some("自分"), text = "test"
  - build_append_args_for_emission_at(&segment, Some(1_000), 1_040, None) を呼ぶ (observed_at_secs を明示的に None で渡す)
- Assert:
  - result.is_some() = true (Some 経路)
  - result.unwrap().1 (offset) == 40 (= stream_started 1040 - session_started 1000)
  - result.unwrap().0 (speaker) == "自分"
  - result.unwrap().2 (text) == "test"
- コメント: 「emission helper 経由でも zero_start + observed=None の場合に stream_started_at_secs フォールバックが効く現契約。Loop 1 T1 (segment_to_append_args 低層) と対称な emission helper 高層の test。」

## T2: build_append_args_for_emission_at_saturates_offset_when_session_after_observed_with_zero_start
- 目的: emission helper 経由で「stream<session かつ zero_start かつ observed=Some(stream-session 範囲外)」のケースで saturating_sub によって offset が 0 に飽和する現契約を固定。clock 逆転を emission helper 経由で確認する境界 test。
- 既存 test (`build_append_args_saturates_offset_when_stream_precedes_session`) は (start_ms=0, end_ms=100) で has_engine_timestamp=true 経路。本 T2 は (zero_start = start_ms=0, end_ms=0) で has_engine_timestamp=false 経路。
- Setup:
  - segment.start_ms = 0, end_ms = 0, speaker = Some("相手側"), text = "early"
  - session_started_at_secs = Some(1_000), stream_started_at_secs = 990, observed_at_secs = Some(995)
  - build_append_args_for_emission_at(&segment, Some(1_000), 990, Some(995)) を呼ぶ
- Assert:
  - result.is_some() = true
  - result.unwrap().1 (offset) == 0 (= 995 - 1000 → saturating_sub で 0 に飽和)
- コメント: 「zero_start segment (has_engine_timestamp=false) で observed_at_secs<session_started_at_secs のとき、saturating_sub によって offset=0 飽和する emission helper 経由の現契約。Loop 1 で範囲外だった zero_start + clock 逆転の組み合わせ。」

## T3: build_append_args_for_emission_at_returns_some_for_zero_start_with_is_error_some_false
- 目的: 3 軸 (is_error=Some(false), zero_start, observed=Some) の組み合わせで Some 経路 + zero_start 計算が動く契約を CI 固定。`is_error.unwrap_or(false)` の Some(false) 分岐 + zero_start segment + observed の 3 軸合流 test。
- 既存 test (`build_append_args_for_emission_returns_some_when_is_error_is_some_false`) は sample_segment (start_ms=2000, engine path) のみ。zero_start path は未保護。
- Setup:
  - segment.start_ms = 0, end_ms = 0, speaker = Some("自分"), text = "realtime", is_error = Some(false) (None ではなく明示的に Some(false))
  - build_append_args_for_emission_at(&segment, Some(1_000), 1_000, Some(1_075)) を呼ぶ
- Assert:
  - result.is_some() = true (Some(false) は早期 return せず Some を返す)
  - result.unwrap().1 (offset) == 75 (= observed 1075 - session 1000)
  - result.unwrap().0 (speaker) == "自分"
  - result.unwrap().2 (text) == "realtime"
- コメント: 「is_error=Some(false) かつ zero_start segment かつ observed=Some の 3 軸合流。Some(false) は unwrap_or(false) で false → 早期 return せず Some を返し、zero_start 経路で observed_at_secs を使う現契約。is_error の Option 3 状態 × zero_start/engine の 2 経路の交差点を保護。」
