## 設計意図 (なぜこれをやるか)
- 既存 3 件の `is_err()` だけでは「Result が Err である」しか保証していない。実装が `ErrorKind::PermissionDenied` を返すように壊れても test は通る。
- `ErrorKind::NotFound` を契約強制すれば、エラー種別の不変条件を type-level で守れる。OS 互換性 (Windows/macOS/Linux で文言は違うが ErrorKind は同じ) を維持しつつ、回帰検知器を強化できる。
- 既存の `assert_eq!(err.kind(), ErrorKind::InvalidInput);` パターンと対称化することで、ファイル内の test 規約を統一する (Tidy First の精神)。
