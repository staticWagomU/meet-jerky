実装条件:
- `TRANSCRIPTION_ERROR_EVENT` の event 名、payload 型ガード、エラー文言、TranscriptDisplay / LiveCaptionWindow、Rust 側、録音 state、音声取得 state、start/stop/restart フローは変更しない。
- `TranscriptView.tsx` の `TRANSCRIPTION_ERROR_EVENT` listener 内で、disposed でない error event を受けたときに `setIsTranscriptionOperationPending(false)` を追加してください。
- payload が不正な場合も、error event 受信として pending は停止側へ倒してください。既存の `setIsTranscribing(false)` と同じ粒度でよいです。
