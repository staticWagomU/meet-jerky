## 追加するテスト 3 件 (関数名はそのまま使う)

### A. `start_with_output_returns_already_active_when_started_via_basic_start`
- 振る舞い: `start("title-A", 1700000000)` → `start_with_output("title-B", 1700000010, tmp_dir, jst())` が `Err(SessionManagerError::AlreadyActive)` を返す
- 不変条件確認:
  - 戻り値が AlreadyActive
  - `is_active() == true`
  - `current_title() == Some("title-A")` (元のタイトルが保たれる)
  - `current_started_at_secs() == Some(1700000000)` (元の started_at が保たれる)
- 設計意図: state machine の対称性。start で AlreadyActive になったら start_with_output でも同じく AlreadyActive を返す。

### B. `start_returns_already_active_when_started_via_start_with_output`
- 振る舞い: `start_with_output("title-A", 1700000000, tmp_dir, jst())` → `start("title-B", 1700000010)` が `Err(SessionManagerError::AlreadyActive)` を返す
- 不変条件確認:
  - 戻り値が AlreadyActive
  - `is_active() == true`
  - `current_title() == Some("title-A")` (元のタイトルが保たれる)
  - `current_started_at_secs() == Some(1700000000)` (元の started_at が保たれる)
- 設計意図: 上記 A の対称テスト。start_with_output 経由の active 状態でも start は AlreadyActive を返す。

### C. `discard_clears_session_started_with_output_and_subsequent_append_returns_not_active`
- 振る舞い: `start_with_output("title", 1700000000, tmp_dir, jst())` → `discard()` Ok → `append("speaker", 0, "text")` が `Err(SessionManagerError::NotActive)` を返す
- 不変条件確認:
  - discard の戻り値が Ok(())
  - 後続 append の戻り値が NotActive
  - `is_active() == false`
  - **ファイルが書かれていないこと** (`list_md_files(tmp_dir).is_empty()` で確認)
- 設計意図: discard は output 設定があっても何も書かずに sweep する。後続の append は NotActive を返す。

## 設計意図 (なぜこれをやるか)
- 既存テストは `start` 経由の AlreadyActive のみ確認、`start_with_output` 経由の AlreadyActive 不変条件は未カバー。
- discard 後の append が NotActive を返すことは間接的に他テストで確認されているが、**output 設定がある状態での discard** は未確認。これが書き込まずに sweep する不変条件を裏付ける。
- state machine の網羅 (start vs start_with_output × discard) は将来 `start` と `start_with_output` の guard logic を統合/分離するリファクタへの安全網となる。
- リスク極小: 既存テストの組み合わせを変えただけで新しい依存関係は不要。
