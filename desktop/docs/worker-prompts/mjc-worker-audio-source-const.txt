# 実装上の注意
- **transcription.rs (line 1052, 1056, 1320, etc) の `"microphone"` / `"system_audio"` 文字列リテラルは触らない** = transcription source 識別子は parse 処理 (CLI/設定入力) 用で event payload 出力とは責務が異なる。同じ文字列値だが意味的責務が違うので、安易に同じ定数で集約すると後で困る。別ループ送り。
- **fully-qualified path 採用** = `use crate::audio_event::AUDIO_SOURCE_MICROPHONE` を追加せず `crate::audio_event::AUDIO_SOURCE_MICROPHONE` を直接書く。理由: tests mod 側で unused_imports 警告経路を増やさない (前 mjc-main-15 Loop 2 と同じ判断)。同モジュール内の `build_audio_drop_event_payload` 呼び出しと一貫したスタイル維持。
- json! macro 内で `"source": MY_CONST` の構文は標準サポート (serde_json 仕様)。
- cfg(target_os = "macos") gate は system_audio.rs の既存構造そのまま維持。
- AUDIO_SOURCE_MICROPHONE / AUDIO_SOURCE_SYSTEM_AUDIO は `pub(crate)` (crate 内のみ公開) で十分。
