## wbxmjs URL 仕様 (Webex Meeting Join Service)

形式: `https://<site>.webex.com/wbxmjs/joinservice/sites/<site>/meeting/<segment>/<token>`

例:
- `https://acme.webex.com/wbxmjs/joinservice/sites/acme/meeting/download/m123abc`
- `https://acme.webex.com/wbxmjs/joinservice/sites/acme/meeting/m123abc`

検知条件 (誤検知防止のため厳格):
1. `is_webex_host(host)` で host が `*.webex.com` または `webex.com` であること (DNS label 検証付き、既存)
2. path が `/wbxmjs/joinservice/sites/` で始まり、その後に **少なくとも site セグメント + meeting セグメント** が続くこと
   - 例: `/wbxmjs/joinservice/sites/acme/meeting/...` (4 segments + meeting キーワード) は accept
   - `/wbxmjs/joinservice/sites/` (空) や `/wbxmjs/joinservice/sites/acme` (meeting 段がない) は reject
3. query 不要 (wbxmjs URL 自体に meeting context が含まれる)

## 実装方針 (TDD Red→Green)

1. **TDD Red**: 失敗テストを 1 件先に追加し、`cargo test --lib` で失敗を確認
   - 例: `classify_meeting_url("https://acme.webex.com/wbxmjs/joinservice/sites/acme/meeting/m123abc")` が `Some(MeetingUrlClassification { service: "Webex", host: "acme.webex.com" })` を返す期待
2. **TDD Green**: 最小実装で pass させる
   - `is_wbxmjs_path(path) -> bool` ヘルパー追加 (path prefix 検証)
   - `is_webex_wbxmjs_meeting_url(host, path) -> bool` 追加 (host + path 両ガード)
   - `classify_meeting_url` の Webex 分岐に OR 条件で追加: `is_webex_meeting_url(...) || is_webex_jphp_meeting_url(...) || is_webex_wbxmjs_meeting_url(...)`
3. **追加テスト** (Red phase 後、Green phase 前に揃える):
   - **accept**: subdomain webex / meeting 段あり (`https://acme.webex.com/wbxmjs/joinservice/sites/acme/meeting/m123abc`)
   - **accept**: subdomain webex / meeting 段 + 追加 token 段 (`https://acme.webex.com/wbxmjs/joinservice/sites/acme/meeting/download/m123abc`)
   - **accept**: trailing slash 許容 (`https://acme.webex.com/wbxmjs/joinservice/sites/acme/meeting/m123abc/`) ※ Personal Room と j.php と整合性
   - **reject**: meeting 段なし (`https://acme.webex.com/wbxmjs/joinservice/sites/acme`)
   - **reject**: path prefix 違反 (`https://acme.webex.com/wbxmjs/foo/bar`)
   - **reject**: 非 webex host (`https://fake-webex.example.com/wbxmjs/joinservice/sites/acme/meeting/m123abc`)
   - **reject**: site segment 空 (`https://acme.webex.com/wbxmjs/joinservice/sites//meeting/m123abc`)
4. **rustdoc 分類ルール更新**: `is_webex_meeting_url` 上のコメントを以下に書き換え:
