背景 / Why this matters:
- meet-jerky は Apple Speech (macOS native) を含む 4 つの transcription engine を持つ。`normalize_segment_text` (private fn, L67-L69) は Apple Speech engine が ffi 経由で受け取った文字列を最終的な `TranscriptionSegment.text` として返す前の最後の正規化処理。
- 既存テスト 4 件: `language_to_locale_maps_supported_languages` / `language_to_locale_falls_back_to_japanese_for_auto_or_unknown` / `normalize_segment_text_trims_edges_only` / `engine_errors_on_non_macos`
- 未カバーの境界仕様:
  1. `normalize_segment_text` は private fn で「`text.trim().to_string()`」だけ。空入力/whitespace-only/内部空白保持の境界が test で固定されていない → 関数を簡素化/別実装に置換するリファクタへの検知装置がない。
  2. `language_to_locale` の case-sensitive 仕様 (`"JA"` や `"En"` は fallback) が test で固定されていない → 将来 case-insensitive 化される誤リファクタを検知できない。
  3. `language_to_locale` の空文字列・whitespace 入力時の fallback 挙動が未確認。
  4. `normalize_segment_text` の Unicode whitespace (tab/newline/全角スペースなど) が trim される仕様が未確認。

追加してほしいテスト (5 件、~10-15 分で完了):

1. `normalize_segment_text_returns_empty_for_empty_input`:
   - `assert_eq!(super::normalize_segment_text(""), "")`
   - Why: 空入力時の挙動を test で固定。`text.trim().to_string()` の `""` 経路を private fn 直接呼び出しで保証。

2. `normalize_segment_text_returns_empty_for_whitespace_only_inputs`:
   - `assert_eq!(super::normalize_segment_text("   "), "")` (半角スペースのみ)
   - `assert_eq!(super::normalize_segment_text("\t\n  "), "")` (tab/newline 混在)
   - `assert_eq!(super::normalize_segment_text(" \t \n "), "")` (混在パターン 2)
   - Why: ASCII whitespace 全種類が `str::trim` で除去される仕様を直接固定。Apple Speech ffi が空白だけの segment を返した場合の最終出力が確実に空文字列になる契約。

3. `normalize_segment_text_preserves_internal_whitespace_only_trims_edges`:
   - `assert_eq!(super::normalize_segment_text("  hello world  "), "hello world")` (前後 trim, 内部空白保持)
   - `assert_eq!(super::normalize_segment_text("  hello   world  "), "hello   world")` (内部の連続空白も保持)
   - `assert_eq!(super::normalize_segment_text("  日本語  テキスト  "), "日本語  テキスト")` (日本語 + 内部空白保持)
   - Why: 既存テスト `normalize_segment_text_trims_edges_only` は単一ケースの assert のみで「内部空白を collapse しない」契約を多角的に確認していない。連続空白を `split_whitespace().join(" ")` のように collapse する誤リファクタを 3 ケースで網羅検知。

4. `language_to_locale_is_case_sensitive_and_falls_back_for_uppercase`:
   - `assert_eq!(super::language_to_locale("JA"), "ja-JP")`
   - `assert_eq!(super::language_to_locale("EN"), "ja-JP")` (大文字 EN は en-US ではなく fallback ja-JP)
   - `assert_eq!(super::language_to_locale("Ja"), "ja-JP")` (mixed case)
   - `assert_eq!(super::language_to_locale("jA"), "ja-JP")` (mixed case 逆)
   - Why: `match language { "ja" => "ja-JP", "en" => "en-US", _ => "ja-JP" }` の case-sensitive 仕様を 4 ケースで明示固定。case-insensitive 化を検討するリファクタが入った場合に、現挙動の意図 (上位層で正規化済み文字列を渡す前提) を test で表現する。

5. `language_to_locale_falls_back_for_empty_and_whitespace_inputs`:
   - `assert_eq!(super::language_to_locale(""), "ja-JP")`
   - `assert_eq!(super::language_to_locale(" "), "ja-JP")` (whitespace は trim せず fallback)
   - `assert_eq!(super::language_to_locale("ja "), "ja-JP")` (trailing space で fallback、trim しない仕様)
   - Why: 既存テストは `"auto"`/`"fr"`/`"unknown"` の fallback のみカバー。空文字列・whitespace 入りの場合も fallback `"ja-JP"` になる契約を固定し、上位層が trim/lowercase を保証すべきという責任境界を明示。
