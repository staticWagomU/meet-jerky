## 対象型

```rust
#[derive(Debug)]
enum AudioCommand {
    Samples(Vec<f32>),
    /// 入力終了を示す。WS タスクは flush してから WS をクローズする。
    Finalize,
}
```

= **private enum** (= `pub` ではないので外部公開されない、ws_task module 内部の制御コマンド用)、`Debug` のみ derive。`Clone` / `PartialEq` / `Default` / `Serialize` / `Deserialize` は **派生なし**。
- `Samples(Vec<f32>)` = tuple variant (Vec<f32> payload、PCM サンプル投入用)
- `Finalize` = unit variant (入力終了通知用)

`mod tests` 内では `use super::*;` 経由で AudioCommand に touch 可能 (private 同一 module 範囲内)。

## 追加する 3 件のテスト

### T1: `audio_command_samples_variant_debug_format_contains_variant_name_and_payload_floats`

`AudioCommand::Samples(vec![1.0_f32, -0.5, 0.0])` の Debug 出力に以下が含まれることを **3 段以上の assert** で確認する。

- variant 名 `"Samples"` を含む
- `1.0` の Debug 表示を含む (Rust の f32 Debug は `1.0` 形式)
- `-0.5` の Debug 表示を含む
- `0.0` の Debug 表示を含む (オプション 4 段目、3 段必須を超える場合は採用)

例:
```rust
#[test]
fn audio_command_samples_variant_debug_format_contains_variant_name_and_payload_floats() {
    let cmd = AudioCommand::Samples(vec![1.0_f32, -0.5, 0.0]);
    let formatted = format!("{cmd:?}");
    assert!(formatted.contains("Samples"), "variant 名: {formatted}");
    assert!(formatted.contains("1.0"), "first sample: {formatted}");
    assert!(formatted.contains("-0.5"), "second sample: {formatted}");
    assert!(formatted.contains("0.0"), "third sample: {formatted}");
}
```

### T2: `audio_command_finalize_variant_debug_format_is_exact_variant_name`

`AudioCommand::Finalize` (= unit variant) の Debug 出力が **完全一致** で `"Finalize"` であることを **2 段以上の assert** で確認する。

- `formatted == "Finalize"` の `assert_eq!`
- `formatted.contains("Finalize")` (補助的 contains)

unit variant は Rust の Debug 仕様で「variant 名のみ」が出力される (括弧やフィールドなし) ことを CI 固定。

例:
```rust
#[test]
fn audio_command_finalize_variant_debug_format_is_exact_variant_name() {
    let cmd = AudioCommand::Finalize;
    let formatted = format!("{cmd:?}");
    assert_eq!(formatted, "Finalize", "完全一致: {formatted}");
    assert!(formatted.contains("Finalize"));
}
```

### T3: `audio_command_samples_with_empty_vec_debug_format_contains_variant_name_and_empty_brackets`

`AudioCommand::Samples(vec![])` (= 空 Vec<f32> payload) の Debug 出力に以下が含まれることを **3 段以上の assert** で確認する。

- variant 名 `"Samples"` を含む
- 空 Vec の Debug 表示 `"[]"` を含む (Rust の Vec<f32> Debug は空時 `[]`)
- variant 名と `[]` が同一文字列内に共存することを「文字列全体形」`"Samples([])"` の `contains` で確認

例:
```rust
#[test]
fn audio_command_samples_with_empty_vec_debug_format_contains_variant_name_and_empty_brackets() {
    let cmd = AudioCommand::Samples(vec![]);
    let formatted = format!("{cmd:?}");
    assert!(formatted.contains("Samples"), "variant 名: {formatted}");
    assert!(formatted.contains("[]"), "空 Vec の Debug 表示: {formatted}");
    assert!(formatted.contains("Samples([])"), "tuple variant 形式: {formatted}");
}
```
