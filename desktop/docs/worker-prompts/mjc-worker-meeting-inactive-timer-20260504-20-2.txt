## 1. `inactive_notification_body` 純粋関数を追加 (test 容易性のため)

`notification_body` 関数 (line 338-342 付近、現行 commit 後の正確な位置は要確認) の **直後** に追加:

```rust
/// inactive 通知の本文を生成する純粋関数。
/// app_name と elapsed_secs を受け取って表示文を整形する。test 容易性のため `show_inactive_notification` から分離。
#[cfg_attr(not(target_os = "macos"), allow(dead_code))]
fn inactive_notification_body(app_name: &str, elapsed_secs: u64) -> String {
    let elapsed_min = elapsed_secs / 60;
    format!(
        "{app_name} が {elapsed_min} 分以上検知されていません。会議が終了している場合は録音と文字起こしを停止してください。"
    )
}
```

## 2. `show_inactive_notification` 関数を追加

`show_notification` 関数 (line 326-335 付近、現行 commit 後の正確な位置は要確認) の **直後** に追加:

```rust
/// 会議アプリが long time 検知されていない場合の inactive 通知を発火する。
/// `show_notification` と同じ Tauri Notification API を使い、本文だけ
/// `inactive_notification_body` で生成する別 body にする。
#[cfg_attr(not(target_os = "macos"), allow(dead_code))]
fn show_inactive_notification(app: &AppHandle, app_name: &str, elapsed_secs: u64) {
    use tauri_plugin_notification::NotificationExt;

    let title = "Meet Jerky";
    let body = inactive_notification_body(app_name, elapsed_secs);

    if let Err(e) = app.notification().builder().title(title).body(&body).show() {
        eprintln!("[app_detection] inactive notification show failed: {e}");
    }
}
```

## 3. `check_all_inactive_bundles()` ヘルパー関数を追加

`check_meeting_inactive_for_bundle` (前 Loop で追加) の **直後** に追加:

```rust
/// `WATCHED_BUNDLE_IDS` 全件に対して `check_meeting_inactive_for_bundle` を呼び、
/// `Some(elapsed)` が返ったら `show_inactive_notification` を発火する。
/// タイマースレッドから定期的に呼ばれる。
#[cfg_attr(not(target_os = "macos"), allow(dead_code))]
fn check_all_inactive_bundles() {
    let state = match STATE.get() {
        Some(s) => s,
        None => return,
    };
    for (bundle_id, app_name) in WATCHED_BUNDLE_IDS {
        if let Some(elapsed) = check_meeting_inactive_for_bundle(bundle_id) {
            show_inactive_notification(&state.app_handle, app_name, elapsed);
        }
    }
}
```

## 4. `start()` 関数内でタイマースレッドを起動

`start()` (line 98-113 付近、現行 commit 後の正確な位置は要確認) の `STATE.set(...).is_ok()` で `first_time` 確認後、`#[cfg(target_os = "macos")] macos::start_detection();` の **直前または直後** に **macOS 限定** でタイマースレッドを起動:

```rust
if first_time {
    #[cfg(target_os = "macos")]
    {
        macos::start_detection();

        // 会議終了検知タイマー: 60 秒周期で全 watched bundle を check し、
        // 必要なら inactive 通知を発火する。
        std::thread::spawn(|| loop {
            std::thread::sleep(std::time::Duration::from_secs(60));
            check_all_inactive_bundles();
        });
    }
}
```

**重要**:
- タイマースレッドは **無限ループ + sleep**。アプリ終了時に明示的な join はせず、デーモン的に動作させる (Tauri アプリ終了時に thread が殺される、これは既存 `std::thread::spawn` パターン (line 701, 736) と同じ)。
- 既存 `macos::start_detection();` の前後どちらでも OK。**`macos::start_detection();` の後** が NSWorkspace 初期化と並走する形で自然。

# 実装上の注意

- **macOS cfg gate 厳守**: タイマースレッド起動は `#[cfg(target_os = "macos")]` で囲む。Linux/Windows では NSWorkspace コールバックが来ないため `last_seen_secs` が永久に空 = タイマーを動かしても無意味なので gate する。
- **既存パターン踏襲**: `std::thread::spawn` は既存 `macos::detection_callback` (line 701) と `browser_url_callback` (line 736) で確立済 = 完全踏襲。
- **lock 順序**: `check_all_inactive_bundles` は wrapper 関数を呼ぶだけで lock を直接取らない (wrapper 内部の lock 順序は前 Loop で確定済) = lock 干渉なし。
- **`#[allow(dead_code)]` 削除の確認**: 削除後に `cargo clippy --all-targets -- -D warnings` で **dead_code warning が再発しない** ことを確認 = 連鎖的に呼び出されている証拠。もし warning が出たら削除しすぎ (=`#[cfg_attr]` 側まで消した可能性) を疑う。
- **show_inactive_notification の test なし**: AppHandle が必要なため unit test 不能 = 純粋関数 `inactive_notification_body` の test で十分とする。**この判断を AGENT_LOG.md の「設計判断」に明記**。
- **frontend event emit は scope 外**: 本 Loop では macOS 通知センター発火のみ。`meeting-inactive-detected` Tauri event 追加は次 Loop (= 将来 F-Loop5 等) に分離。
- **担当範囲外を編集しない** (lib.rs / 他のファイル / Swift bridge は触らない)。
- **コミット禁止**。実装と検証だけ。
