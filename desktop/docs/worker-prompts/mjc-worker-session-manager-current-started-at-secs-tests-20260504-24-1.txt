# 追加する test 3 件 (Tidy First、ユーザー観測不可、振る舞い不変)

## test 1: `current_started_at_secs_returns_none_when_idle_and_after_finalize_and_after_discard`
- **目的**: lifecycle 軸の None 契約を 3 状態 (never-started / after finalize / after discard) で対称的に CI 固定
- **手順**:
  1. `let manager = SessionManager::new();` → `assert_eq!(manager.current_started_at_secs(), None, "未開始時は None")`
  2. `manager.start("meeting".into(), 100).expect("start");` → `manager.finalize(200).expect("finalize");` → `assert_eq!(manager.current_started_at_secs(), None, "finalize 後は None")`
  3. `manager.start("meeting2".into(), 300).expect("restart");` → `manager.discard().expect("discard");` → `assert_eq!(manager.current_started_at_secs(), None, "discard 後は None")`
- **意図**: stale started_at 値が finalize/discard 後に漏れない契約を CI 固定。既存 test 群は `is_active() == false` のみ確認で `current_started_at_secs() == None` の直接 assert は皆無。

## test 2: `current_started_at_secs_returns_zero_when_session_started_at_unix_epoch_zero`
- **目的**: 値域軸の左境界 (started_at=0、unix epoch) が原値のまま返る契約を CI 固定
- **手順**:
  1. `let manager = SessionManager::new();`
  2. `manager.start("epoch_session".into(), 0).expect("start with started_at=0 should succeed");`
  3. `assert_eq!(manager.current_started_at_secs(), Some(0), "started_at=0 (unix epoch) は Some(0) として原値で返るはず");`
- **意図**: 「if started_at == 0 { None }」のような『0 を unset として扱う』防衛的最適化への誤改修を検知する装置。
- **assert メッセージ**: 「started_at=0 (unix epoch) は Some(0) として原値で返るはず。0 を unset として扱う防衛的最適化を CI で遮断する。」

## test 3: `current_started_at_secs_returns_u64_max_when_session_started_at_far_future`
- **目的**: 値域軸の右境界 (started_at=u64::MAX、極端値) が truncation なく原値のまま返る契約を CI 固定
- **手順**:
  1. `let manager = SessionManager::new();`
  2. `manager.start("future_session".into(), u64::MAX).expect("start with u64::MAX should succeed");`
  3. `assert_eq!(manager.current_started_at_secs(), Some(u64::MAX), "started_at=u64::MAX は Some(u64::MAX) として truncation なく返るはず");`
- **意図**: 内部で `started_at as i64` cast 等を入れた場合に signed overflow で負値化するような誤改修を検知する装置。
- **assert メッセージ**: 「started_at=u64::MAX は truncation なく返るはず。`as i64` cast 等の符号付き truncation を CI で遮断する。」
