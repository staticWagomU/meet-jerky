## 指示内容: 「対称的補強の 3 軸構造」+「Debug 軸補強」6 連続 application = TranscriptionSource enum

`TranscriptionSource` は `src-tauri/src/transcription.rs` l.19-23 で以下の通り定義された enum:

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize)]
#[serde(rename_all = "snake_case")]
pub enum TranscriptionSource {
    Microphone,
    SystemAudio,
}
```

これは mjc-main-30 Loop 1 で application した `SecretKey` enum (Debug + Copy + kebab-case format の 3 軸) と完全に対称な enum 形態。SecretKey で確立した 3 軸構造を TranscriptionSource enum (snake_case format) で再 application する **enum 派生対称軸補強** パターン。

以下 3 つの test を `mod tests` 末尾 (= 既存最終 test = Loop 1 で追加した `transcription_segment_serde_serializes_with_camel_case_field_names_and_skips_none_options` 直後の `}` の **直前**) に追加せよ。

### T1: `transcription_source_debug_output_contains_each_variant_name_per_variant`
**契約**: `format!("{:?}", X)` で各 variant の Debug 出力が variant 名 (`Microphone` / `SystemAudio`) を含み、両 variant の Debug 出力が異なることを CI 固定する。
**注意**: Rust の enum variant の `#[derive(Debug)]` 派生は variant 名のみ出力する (enum 名は出ない)。例: `format!("{:?}", TranscriptionSource::Microphone)` → `"Microphone"` のみ。
**test 設計**:
- `let mic = TranscriptionSource::Microphone;`
- `let sys = TranscriptionSource::SystemAudio;`
- `let dbg_mic = format!("{:?}", mic);`
- `let dbg_sys = format!("{:?}", sys);`
- `assert!(dbg_mic.contains("Microphone"), "Microphone variant の Debug 出力に variant 名 Microphone が含まれる: {dbg_mic}")`
- `assert!(dbg_sys.contains("SystemAudio"), "SystemAudio variant の Debug 出力に variant 名 SystemAudio が含まれる: {dbg_sys}")`
- `assert_ne!(dbg_mic, dbg_sys, "Microphone と SystemAudio の Debug 出力は異なる")`
- これにより Debug 派生が消されたり、`#[derive(Debug)]` を手動実装に置き換えて variant 名を隠蔽する誤改修を遮断する。

### T2: `transcription_source_copy_trait_keeps_original_usable_after_assignment`
**契約**: `#[derive(Copy)]` 派生で `let copied = original;` の後も `original` が引き続き使える契約 (Copy semantics) を CI 固定する。Copy が外されれば move semantics になり、`original` を後続で使うとコンパイルエラーになるため、test 自体がコンパイルできなくなることで誤改修を遮断する。
**test 設計**:
- `let original = TranscriptionSource::Microphone;`
- `let copied = original;` (Copy が無いと move される)
- `assert_eq!(original, copied, "Copy 派生で copied が original の値と等しい");`
- `assert_eq!(original, TranscriptionSource::Microphone, "Copy 後も original は Microphone のまま使える");` (original が move されていれば**コンパイルエラーになる**ため、CI で誤改修を検出)
- SystemAudio variant でも同様:
  - `let s_original = TranscriptionSource::SystemAudio;`
  - `let s_copied = s_original;`
  - `assert_eq!(s_original, s_copied, "Copy 派生で SystemAudio も copy される");`
  - `assert_eq!(s_original, TranscriptionSource::SystemAudio, "Copy 後も SystemAudio の original は使える");`
- これにより `#[derive(Copy)]` が外されて move semantics に変わる誤改修を遮断する。

### T3: `transcription_source_serde_serializes_each_variant_with_snake_case_value`
**契約**: `#[serde(rename_all = "snake_case")]` 規則で各 variant の Serialize value が snake_case 文字列になる契約を **for ループで全 variant 走査** で CI 固定する (拡張追従型 test 設計)。Microphone → `"microphone"`, SystemAudio → `"system_audio"`。
**test 設計**:
- 期待値の配列を作る: `let cases: &[(TranscriptionSource, &str)] = &[(TranscriptionSource::Microphone, "microphone"), (TranscriptionSource::SystemAudio, "system_audio")];`
- for ループで走査:
  - `for (variant, expected) in cases {`
  - `    let value = serde_json::to_value(variant).unwrap();`
  - `    assert_eq!(value, serde_json::Value::String((*expected).to_string()), "{:?} は serde で {} に serialize される", variant, expected);`
  - `    let s = serde_json::to_string(variant).unwrap();`
  - `    assert_eq!(s, format!("\"{}\"", expected), "{:?} は serde で \"{}\" に文字列化される", variant, expected);`
  - `}`
- これにより `#[serde(rename_all = "snake_case")]` が外される / 個別 rename される / 別 case (camelCase / kebab-case) に変更される誤改修を遮断する = 拡張追従型 (将来 variant 追加時に配列に追加するだけで自動カバー) = mjc-main-30 Loop 1 (SecretKey の kebab-case format 軸) と対称。
