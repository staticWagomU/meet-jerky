追加してほしいテスト (6 件、~15 分で完了):

**追加先**: `mod ws_task { ... #[cfg(test)] mod pending_timeout_tests { ... } }` の **`mod pending_timeout_tests` 末尾** に追加。
- `mod ws_task` 外側からは private fn にアクセスできないため、必ず `mod ws_task` 内側のテスト mod に追加すること。
- 既存 `mod pending_timeout_tests` 内では `super::*` または `super::is_scribe_error_event` 等で private fn にアクセス可能。

1. `is_scribe_error_event_returns_true_for_scribe_prefix_and_error_suffix`:
   - `assert!(super::is_scribe_error_event("scribe_validation_error"))` (典型ケース)
   - `assert!(super::is_scribe_error_event("scribe_audio_error"))` (typical)
   - `assert!(super::is_scribe_error_event("scribe_some_long_underscore_error"))` (内部 underscore 多数)
   - Why: AND 条件の両方マッチ pattern が壊れた瞬間に検知。

2. `is_scribe_error_event_returns_false_when_either_prefix_or_suffix_missing`:
   - `assert!(!super::is_scribe_error_event("scribe_audio"))` (suffix なし)
   - `assert!(!super::is_scribe_error_event("audio_error"))` (prefix なし)
   - `assert!(!super::is_scribe_error_event(""))` (空文字列)
   - `assert!(!super::is_scribe_error_event("non_scribe_error"))` (prefix が "non_" で始まる)
   - `assert!(!super::is_scribe_error_event("SCRIBE_ERROR"))` (大文字: case-sensitive 仕様確認)
   - Why: AND 短絡条件と case-sensitive 仕様を 5 ケースで明示。

3. `extract_transcript_prefers_transcript_field_over_text_field`:
   - `let value = serde_json::json!({ "transcript": "hello", "text": "ignored" });`
   - `assert_eq!(super::extract_transcript(&value), Some("hello".to_string()))`
   - 別 case: `let value = serde_json::json!({ "text": "fallback only" });`
   - `assert_eq!(super::extract_transcript(&value), Some("fallback only".to_string()))`
   - Why: ElevenLabs API レスポンスの両 field 共存時の優先順位 (`transcript` first) を固定。順序リファクタを即座に検知。

4. `extract_transcript_falls_back_to_words_array_with_text_or_word_keys`:
   - `let value = serde_json::json!({ "words": [{"text": "hello"}, {"word": " world"}, {"text": "!"}] });`
   - `assert_eq!(super::extract_transcript(&value), Some("hello world!".to_string()))` (text と word 混在で順序保持・空文字列 join)
   - 別 case: `let value = serde_json::json!({ "words": [] });`
   - `assert_eq!(super::extract_transcript(&value), Some("".to_string()))` (空 array → 空 string)
   - 別 case: `let value = serde_json::json!({ "words": [{"unrelated": "x"}] });`
   - `assert_eq!(super::extract_transcript(&value), Some("".to_string()))` (text/word ともになければ filter_map で skip → 空 string)
   - Why: words array の text/word 両 key fallback と filter_map による空文字列 join を 3 ケースで網羅。

5. `extract_transcript_returns_none_when_no_relevant_field_present`:
   - `let value = serde_json::json!({});` (空 object)
   - `assert_eq!(super::extract_transcript(&value), None)`
   - 別 case: `let value = serde_json::json!({ "unrelated": "x", "other": 42 });`
   - `assert_eq!(super::extract_transcript(&value), None)`
   - 別 case: `let value = serde_json::json!({ "transcript": 123 });` (transcript が string でない場合)
   - `assert_eq!(super::extract_transcript(&value), None)` (as_str() が None → 次の優先順位 text もない → None)
   - Why: 全 fallback path が exhaust された時の None 返却契約。type 違い (number) で as_str() が失敗するケースも明示。

6. `extract_error_message_traverses_three_priority_paths`:
   - case 1: `let value = serde_json::json!({ "message": "top-level message", "error": "ignored" });`
   - `assert_eq!(super::extract_error_message(&value), Some("top-level message".to_string()))` (message 優先)
   - case 2: `let value = serde_json::json!({ "error": "string-error" });`
   - `assert_eq!(super::extract_error_message(&value), Some("string-error".to_string()))` (error が string)
   - case 3: `let value = serde_json::json!({ "error": { "message": "nested" } });`
   - `assert_eq!(super::extract_error_message(&value), Some("nested".to_string()))` (error が object → message を取り出す)
   - case 4: `let value = serde_json::json!({});`
   - `assert_eq!(super::extract_error_message(&value), None)` (全 field 不在)
   - Why: 3 段 fallback (`message` → `error:string` → `error.message:object`) を 4 ケースで網羅。ElevenLabs API のレスポンス形式バリエーション吸収 layer の不変条件。
