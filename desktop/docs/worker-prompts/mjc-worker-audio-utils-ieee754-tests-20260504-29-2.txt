## 追加する test 3 件 (命名と意図を厳守)

### T1: `sanitize_audio_sample_passes_through_negative_zero_preserving_bit_pattern`

- 目的: `-0.0` (IEEE 754 で `+0.0` と value-equal だが bit pattern が違う) も passthrough する現契約を CI 固定。`is_finite()` は `-0.0` で true、`clamp(-1.0, 1.0)` は `-0.0` を `-0.0` のまま返す性質を test 化。
- 実装:
  ```rust
  let result = sanitize_audio_sample(-0.0_f32);
  assert_eq!(result, 0.0_f32);
  // bit pattern まで保護: -0.0 と +0.0 は == では区別されないため、to_bits で確認
  assert_eq!(result.to_bits(), (-0.0_f32).to_bits(),
      "negative zero must preserve bit pattern (sign bit) through sanitize");
  ```

### T2: `sanitize_audio_sample_clamps_f32_max_to_one_and_f32_min_finite_to_minus_one`

- 目的: `f32::MAX` (≈ 3.4e38、有限値の最大) は 1.0 にクランプ、`f32::MIN` (≈ -3.4e38、有限値の最小、`f32::MIN` は `-f32::MAX` と等価) は -1.0 にクランプする現契約を CI 固定。`is_finite()` は両方 true (Inf ではない)、`clamp` で範囲外を畳み込む。
- 実装:
  ```rust
  assert_eq!(sanitize_audio_sample(f32::MAX), 1.0,
      "f32::MAX is finite (not Inf) so clamp must reduce to 1.0");
  assert_eq!(sanitize_audio_sample(f32::MIN), -1.0,
      "f32::MIN (= -f32::MAX) is finite (not -Inf) so clamp must reduce to -1.0");
  ```

### T3: `sanitize_audio_sample_passes_through_subnormal_min_positive_without_flush_to_zero`

- 目的: `f32::MIN_POSITIVE` (≈ 1.18e-38、subnormal の最小正値の境界 = normal の最小正値) は範囲 `[-1.0, 1.0]` 内なので passthrough する現契約を CI 固定。subnormal が flush-to-zero されない (= clamp が subnormal を保持する) 性質を test 化。
- 実装:
  ```rust
  let subnormal = f32::MIN_POSITIVE;
  let result = sanitize_audio_sample(subnormal);
  assert_eq!(result, subnormal,
      "f32::MIN_POSITIVE (smallest normal positive) is in [-1.0, 1.0] so must passthrough without flush-to-zero");
  ```
