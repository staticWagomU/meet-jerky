## webappng URL 仕様 (Webex Web App Next Generation)

形式: `https://<site>.webex.com/webappng/sites/<site>/meeting/info/<token>`

例:
- `https://acme.webex.com/webappng/sites/acme/meeting/info/m123abc`
- `https://acme.webex.com/webappng/sites/acme/meeting/download/m123abc` (将来別 action 段の可能性、本ループでは `info` のみ accept)

検知条件 (誤検知防止のため厳格):
1. `is_webex_host(host)` で host が `*.webex.com` または `webex.com` であること (DNS label 検証付き、既存)
2. path が `/webappng/sites/<site>/meeting/info/...` 形式であること
   - prefix `/webappng/sites/` で始まる
   - 次に site segment (非空)
   - 次に `meeting` キーワード
   - 次に `info` action segment ※ wbxmjs (任意 action 段) よりさらに厳格
   - その後に最低 1 つの token segment (非空)
3. query 不要

## 実装方針 (TDD Red→Green)

1. **TDD Red**: 失敗テストを 1 件先に追加し、`cargo test --lib` で失敗を確認
   - 例: `classify_meeting_url("https://acme.webex.com/webappng/sites/acme/meeting/info/m123abc")` が `Some(MeetingUrlClassification { service: "Webex", host: "acme.webex.com" })` を返す期待
2. **TDD Green**: 最小実装で pass させる
   - `is_webappng_path(path) -> bool` ヘルパー追加 (path prefix 検証 + segment 構造検証)
   - `is_webex_webappng_meeting_url(host, path) -> bool` 追加 (host + path 両ガード)
   - `classify_meeting_url` の Webex 分岐に OR 条件で追加: `... || is_webex_webappng_meeting_url(...)`
3. **追加テスト** (Red phase 後、Green phase 前に揃える):
   - **accept**: subdomain webex / 完全形 (`https://acme.webex.com/webappng/sites/acme/meeting/info/m123abc`)
   - **accept**: trailing slash 許容 (`https://acme.webex.com/webappng/sites/acme/meeting/info/m123abc/`) ※既存 Personal Room / j.php / wbxmjs と整合性
   - **reject**: token segment なし (`https://acme.webex.com/webappng/sites/acme/meeting/info`) ※ wbxmjs は accept だったが webappng は token 必須
   - **reject**: action segment が `info` 以外 (`https://acme.webex.com/webappng/sites/acme/meeting/download/m123abc`)
   - **reject**: `meeting` キーワード違反 (`https://acme.webex.com/webappng/sites/acme/foo/info/m123abc`)
   - **reject**: 非 webex host (`https://fake-webex.example.com/webappng/sites/acme/meeting/info/m123abc`)
   - **reject**: site segment 空 (`https://acme.webex.com/webappng/sites//meeting/info/m123abc`)
4. **rustdoc 分類ルール更新**: `is_webex_meeting_url` 上のコメントを以下に書き換え:
   ```
   // Personal Room (`/meet/<id>`) / j.php 招待 URL (`/<site>/j.php?MTID=<token>`) /
   // wbxmjs Meeting Join Service URL (`/wbxmjs/joinservice/sites/<site>/meeting/...`) /
   // webappng URL (`/webappng/sites/<site>/meeting/info/<token>`) に対応。
   // 他の Webex URL 形式は誤検知防止のため将来課題。
   ```
   (※「将来課題」リストから webappng を削除、対応済みリストに移動 = Webex 招待 URL 主要 4 系統網羅完了)
