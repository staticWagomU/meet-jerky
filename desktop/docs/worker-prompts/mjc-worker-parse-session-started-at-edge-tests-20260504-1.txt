# 関数仕様 (再掲)

```rust
fn parse_session_started_at_secs(stem: &str) -> Option<u64> {
    let prefix = stem.split('-').next().unwrap_or("");
    let started_at_secs = prefix.parse::<u64>().ok()?;
    if started_at_secs > MAX_JS_DATE_UNIX_SECS {
        return None;
    }
    Some(started_at_secs)
}
```

`MAX_JS_DATE_UNIX_SECS = 8_640_000_000_000` (L20)。

# 追加すべき #[test] 関数 (8 件)

各 test は単体で意味を持ち、関数名は WHAT を示すこと。

## A. `parse_session_started_at_secs_returns_none_for_empty_string`
- 入力 `""` → `split('-').next()` は `Some("")` → `"".parse::<u64>()` → Err → `None`。
- 期待: `assert_eq!(parse_session_started_at_secs(""), None);`

## B. `parse_session_started_at_secs_accepts_pure_numeric_without_hyphen`
- 入力 `"1234567890"` → split で先頭 `"1234567890"` → 1234567890。
- 期待: `Some(1234567890)`。

## C. `parse_session_started_at_secs_accepts_numeric_with_suffix_after_hyphen`
- 入力 `"1234567890-meet-abc"` → split で先頭 `"1234567890"` → 1234567890。
- 期待: `Some(1234567890)` (現実的なファイル名形式)。

## D. `parse_session_started_at_secs_returns_none_for_leading_hyphen`
- 入力 `"-1234567890"` → split で先頭 `""` → parse 失敗 → `None`。
- 期待: `None`。

## E. `parse_session_started_at_secs_returns_none_for_overflow_u64`
- 入力 `"99999999999999999999999"` (u64 の上限 18446744073709551615 を超える) → parse 失敗 → `None`。
- 期待: `None`。

## F. `parse_session_started_at_secs_returns_none_for_negative_sign_prefix`
- 入力 `"-12345"` → 上記 D と同じ理由で `None`。
- D と入力が異なるが意図は別なので別 test として分離 (D は「ハイフン始まり」の構文ケース、F は「負号として扱われない」ことを示す意図)。
- 期待: `None`。

## G. `parse_session_started_at_secs_accepts_boundary_max`
- 入力 `"8640000000000"` (= `MAX_JS_DATE_UNIX_SECS`) → `8_640_000_000_000` → `> MAX` ではなく `== MAX` なので Some。
- 期待: `Some(8_640_000_000_000)`。

## H. `parse_session_started_at_secs_returns_none_just_above_boundary`
- 入力 `"8640000000001"` (= `MAX_JS_DATE_UNIX_SECS + 1`) → > MAX → `None`。
- 期待: `None`。
