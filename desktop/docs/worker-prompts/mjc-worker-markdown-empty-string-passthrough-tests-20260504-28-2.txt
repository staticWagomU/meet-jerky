# 背景 = mjc-main-20260504-28 Loop 2 = 別ファイル切替 = markdown.rs 空文字 passthrough 契約 完全網羅

`markdown.rs` の `format_session_markdown` (l.24-49) は 14 件の test を持つが、SessionMeta.title / SessionSegment.speaker / SessionSegment.text の 3 つの主要 String フィールドの **空文字 passthrough 契約** が未保護:

1. **title 軸**: `meta.title = ""` の場合、ヘッダは `# {} - {}\n` 形式で空文字を取り込み `#  - <started_at>\n` (`#` の後に 2 連続 space) を出力する現契約。空 title でも segment は出力される。
2. **speaker 軸**: `segment.speaker = ""` の場合、segment 行は `**[<ts>] :** <text>  ` 形式で空 speaker を保持する現契約。空 speaker でも segment は出力される。
3. **text 軸**: `segment.text = ""` の場合、segment 行は `**[<ts>] <speaker>:**   ` 形式で trailing 2 spaces を維持する現契約。**CommonMark の hard break (改行記号) は trailing 2 spaces で表現されるため、空 text でも 2 spaces は省略しない**。

これらは「format_session_markdown が空文字フィールドを passthrough する責務」「上位層 (transcript_bridge / session_store) で空文字フィルタするべきという責務分離」を executable specification として固定する。

## T1: format_session_markdown_with_empty_title_in_meta_produces_header_with_double_space

**目的**: `meta.title = ""` でもヘッダを `#  - <started_at>\n` 形式 (`# ` + 空 title + ` - ` + started_at + `\n` = `#  - 2026-04-17 14:30\n`) で出力する現契約を CI 固定。「空 title だから skip」「空 title だから別 fallback (e.g. `# (untitled)`) を入れる」誤改修を遮断する装置。

**構造**:
```rust
#[test]
fn format_session_markdown_with_empty_title_in_meta_produces_header_with_double_space() {
    let meta = SessionMeta {
        title: String::new(),
        started_at_display: "2026-04-17 14:30".to_string(),
    };
    let segments: Vec<SessionSegment> = Vec::new();

    let expected = "#  - 2026-04-17 14:30\n";
    assert_eq!(
        format_session_markdown(&meta, &segments),
        expected,
        "空 title でも `# ` + 空 + ` - ` + started_at の format を維持する passthrough 契約"
    );
}
```

## T2: format_session_markdown_with_empty_speaker_in_segment_preserves_segment_layout

**目的**: `segment.speaker = ""` でも segment 行を `**[<ts>] :** <text>  ` 形式 (空 speaker + `:` + space + text + trailing 2 spaces) で出力する現契約を CI 固定。「空 speaker だから segment 全体を skip」「空 speaker だから別 fallback (e.g. "(unknown)") を入れる」誤改修を遮断する装置。

**構造**:
```rust
#[test]
fn format_session_markdown_with_empty_speaker_in_segment_preserves_segment_layout() {
    let meta = SessionMeta {
        title: "title".to_string(),
        started_at_display: "2026-04-17 14:30".to_string(),
    };
    let segments = vec![SessionSegment {
        speaker: String::new(),
        timestamp_display: "14:30:05".to_string(),
        text: "hello".to_string(),
    }];

    let expected = "# title - 2026-04-17 14:30\n\n**[14:30:05] :** hello  ";
    assert_eq!(
        format_session_markdown(&meta, &segments),
        expected,
        "空 speaker でも `**[<ts>] :** <text>  ` の format を維持する passthrough 契約"
    );
}
```

## T3: format_session_markdown_with_empty_text_in_segment_preserves_trailing_two_spaces

**目的**: `segment.text = ""` でも segment 行を `**[<ts>] <speaker>:**   ` 形式 (speaker + `:` + space + 空 text + **trailing 2 spaces**) で出力する現契約を CI 固定。CommonMark の hard break (改行記号) は trailing 2 spaces で表現されるため、空 text でも 2 spaces は省略されない。「空 text だから segment 全体を skip」「空 text だから trailing 2 spaces を省略」誤改修を遮断する装置。

**構造**:
```rust
#[test]
fn format_session_markdown_with_empty_text_in_segment_preserves_trailing_two_spaces() {
    let meta = SessionMeta {
        title: "title".to_string(),
        started_at_display: "2026-04-17 14:30".to_string(),
    };
    let segments = vec![SessionSegment {
        speaker: "speaker".to_string(),
        timestamp_display: "14:30:05".to_string(),
        text: String::new(),
    }];

    let expected = "# title - 2026-04-17 14:30\n\n**[14:30:05] speaker:**   ";
    assert_eq!(
        format_session_markdown(&meta, &segments),
        expected,
        "空 text でも trailing 2 spaces (CommonMark hard break) を維持する passthrough 契約"
    );
    assert!(
        format_session_markdown(&meta, &segments).ends_with("  "),
        "空 text でも末尾 2 spaces を保持: hard break 規約の executable specification"
    );
}
```

注意: T3 の expected 文字列の末尾は `**[14:30:05] speaker:**` + space + 空 text + **trailing 2 spaces** = `**[14:30:05] speaker:**   ` (`:` の後に space 1 個 + trailing 2 spaces で計 3 spaces)。Rust の raw string 化を避け、明示的にエスケープなしで `"...**[14:30:05] speaker:**   "` (末尾 3 spaces) と書く。
