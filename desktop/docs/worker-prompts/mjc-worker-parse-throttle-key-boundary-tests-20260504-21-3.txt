# 追加する test (4 件、最小ケース)

mod tests 末尾 (= 既存 `parse_throttle_key_to_display_name_returns_none_for_browser_key_with_empty_service` の **直後**) に以下 4 件追加:

1. `parse_throttle_key_to_display_name_returns_service_for_browser_key_with_colon_in_host`
   - 契約: `"browser:com.apple.Safari:Google Meet:meet.google.com:8443"` (host に `:` ポート番号付き) → `Some("Google Meet")` を返す
   - 設計意図: `splitn(3, ':')` で host 内に `:` が含まれても 3 セグメント目以降は `parts[2]` にまとめられるため、service は `parts[1]` で正しく取れる
   - assertion: `assert_eq!(parse_throttle_key_to_display_name("browser:com.apple.Safari:Google Meet:meet.google.com:8443"), Some("Google Meet".to_string()));`

2. `parse_throttle_key_to_display_name_returns_none_for_browser_prefix_only`
   - 契約: `"browser:"` (prefix のみで残り空) → `None`
   - 設計意図: `strip_prefix("browser:")` 後に空文字 `""` が残り `splitn(3, ':')` で 1 要素 (`[""]`) のみ → `parts.len() < 2` で None
   - assertion: `assert_eq!(parse_throttle_key_to_display_name("browser:"), None);`

3. `parse_throttle_key_to_display_name_returns_none_for_window_title_prefix_only`
   - 契約: `"window-title:"` (prefix のみで残り空) → `None`
   - 設計意図: `strip_prefix("window-title:")` 後に空文字が残り splitn(2, ':') で 1 要素 → parts.len() < 2 で None
   - assertion: `assert_eq!(parse_throttle_key_to_display_name("window-title:"), None);`

4. `parse_throttle_key_to_display_name_returns_none_for_browser_key_with_only_bundle_id`
   - 契約: `"browser:com.apple.Safari"` (bundle_id のみで service 欠落) → `None`
   - 設計意図: `strip_prefix("browser:")` 後に `"com.apple.Safari"` が残り `splitn(3, ':')` で 1 要素 (`["com.apple.Safari"]`) のみ → `parts.len() < 2` で None
   - assertion: `assert_eq!(parse_throttle_key_to_display_name("browser:com.apple.Safari"), None);`

各 test は 2-3 行で完結。doc コメント (`//`) で「契約: <何が固定されるか>」を 1 行書く。
