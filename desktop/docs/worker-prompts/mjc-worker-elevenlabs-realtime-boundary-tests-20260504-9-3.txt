# 1. 背景 / Why

## 対象 1: `is_scribe_error_event` (line 373-375)
```rust
fn is_scribe_error_event(event_name: &str) -> bool {
    event_name.starts_with("scribe_") && event_name.ends_with("_error")
}
```

既存 2 件 (line 549-565):
- `_returns_true_for_scribe_prefix_and_error_suffix`: "scribe_validation_error", "scribe_audio_error", "scribe_some_long_underscore_error"
- `_returns_false_when_either_prefix_or_suffix_missing`: "scribe_audio", "audio_error", "", "non_scribe_error", "SCRIBE_ERROR"

**確認した gap**:
1. **"scribe_error" boundary jets** (中間 0 長): `_` が 1 個で prefix の最後と suffix の最初を兼ねる最短形。`starts_with("scribe_")` true、`ends_with("_error")` true → 結果 true。これは「最短ジャスト」boundary。
2. **"scribe__error" 中間 1 長 (`__`)**: `_` が 2 個で prefix と suffix が独立の `_` を持つ最短形。これも true。
3. **"scribe_x_error" 中間 1 文字**: 既存「scribe_audio_error」より短い 1 文字中間。未確認境界。

## 対象 2: `extract_error_message` (line 414-426)
```rust
fn extract_error_message(value: &Value) -> Option<String> {
    value
        .get("message")
        .and_then(|v| v.as_str())
        .or_else(|| value.get("error").and_then(|v| v.as_str()))
        .or_else(|| {
            value
                .get("error")
                .and_then(|v| v.get("message"))
                .and_then(|v| v.as_str())
        })
        .map(str::to_string)
}
```

既存 1 件 (line 608-630): `_traverses_three_priority_paths`
- priority1: `{ "message": "top-level message", "error": "ignored" }` → "top-level message"
- priority2: `{ "error": "string-error" }` → "string-error"
- priority3: `{ "error": { "message": "nested" } }` → "nested"
- empty: `{}` → None

**確認した gap**:
1. **priority1 が non-string (null/number/array)**: `and_then(|v| v.as_str())` で None になるが `.or_else` で priority2 に fallback する現契約。既存 test では「priority1 が文字列として存在」のみ。
2. **priority2 が non-string (null/number/array) で priority3 が string**: priority2 失敗 → priority3 nested fallback の現契約。既存 test では priority2 と priority3 の単独確認のみ。
3. **priority3 nested message が non-string**: `error: { "message": null }` のような nested 内 non-string → None。既存 test では priority3 が string の場合のみ。
4. **すべての priority が non-string**: 一貫して None 返却の総合契約。

これら 5 件で string 判定 (`as_str()`) の or_else fallback chain と is_scribe_error_event の boundary jets を完全に固定する。

# 2. 追加する 5 件のテスト案 (assertion レベル)

## T1. `is_scribe_error_event_returns_true_for_minimal_overlap_scribe_error`
- 入力 1: `"scribe_error"` (12 chars: prefix と suffix が中央の `_` を共有、最短形のように見えるが実際は `scribe_` (7 chars) + `_error` (6 chars) で個別マッチ)
- assert: `assert!(super::is_scribe_error_event("scribe_error"))`
- 補助 assert: `assert!(super::is_scribe_error_event("scribe_x_error"))` (中間 1 文字、既存最短「scribe_audio」より短い境界)
- 意図: prefix と suffix が **個別独立に** マッチする現契約 (overlap 検出ロジックなし) を固定。将来「中間ゼロ長を排除する」防御強化への誤リファクタを検知。

## T2. `is_scribe_error_event_returns_true_for_zero_length_middle_scribe_double_underscore_error`
- 入力: `"scribe__error"` (中間 0 長で `_` が独立、`scribe_` + `_error` を真に独立に満たす最短形)
- assert: `assert!(super::is_scribe_error_event("scribe__error"))`
- 補助 assert (短い中間 + 末尾近接): `assert!(super::is_scribe_error_event("scribe_a_error"))` (中間「a」1 文字 + 独立 `_`)
- 補助 assert (negative side): `assert!(!super::is_scribe_error_event("scribe"))` (suffix なし、6 chars 単体) と `assert!(!super::is_scribe_error_event("_error"))` (prefix なし、6 chars suffix only)
- 意図: 中間 0 長 (`__`) も true を返す `&&` の最小条件を固定。短い独立形 negative も並べて両端確認。

## T3. `extract_error_message_falls_back_to_priority2_when_priority1_message_is_non_string`
- 入力 1: `{ "message": null, "error": "fallback string" }` → priority1 の null は as_str() で None、or_else で priority2 が "fallback string"
- assert: `assert_eq!(super::extract_error_message(&value), Some("fallback string".to_string()))`
- 入力 2: `{ "message": 123, "error": "from-error" }` → priority1 の number は as_str() で None、priority2 fallback
- assert: `assert_eq!(super::extract_error_message(&value), Some("from-error".to_string()))`
- 入力 3: `{ "message": [], "error": "array-message-skip" }` → priority1 の array は as_str() で None、priority2 fallback
- assert: `assert_eq!(super::extract_error_message(&value), Some("array-message-skip".to_string()))`
- 意図: priority1 が non-string でも sigh fallback で priority2 へ進む現契約を固定。「priority1 が存在すれば priority2 をスキップする」誤実装への誤リファクタを検知。

## T4. `extract_error_message_falls_back_to_priority3_when_priority2_error_is_non_string`
- 入力 1: `{ "error": { "message": "deep nested" } }` → priority2 (error が object なので as_str() で None) → priority3 nested message が string で勝つ
- assert: `assert_eq!(super::extract_error_message(&value), Some("deep nested".to_string()))`
- 入力 2: `{ "message": null, "error": { "message": "deep with priority1 null" } }` → priority1 null + priority2 None (object) + priority3 nested string
- assert: `assert_eq!(super::extract_error_message(&value), Some("deep with priority1 null".to_string()))`
- 補助 assert (priority2 null で priority3 もない場合 None): `{ "error": null }` → priority2 と priority3 両方 None
- assert: `assert_eq!(super::extract_error_message(&value), None)`
- 意図: priority chain 全体が 1 階層ずつ as_str() で判定する現契約を固定。priority2 が object/null でも priority3 へ正しく fallback する fallback chain の総合性を保護。

## T5. `extract_error_message_returns_none_when_all_priorities_yield_non_string`
- 入力 1: `{ "message": null, "error": null }` → priority1 null + priority2 null + priority3 (error.message) None → None
- assert: `assert_eq!(super::extract_error_message(&value), None)`
- 入力 2: `{ "message": [], "error": {} }` → priority1 array + priority2 object + priority3 (error.message) None (empty object に message なし) → None
- assert: `assert_eq!(super::extract_error_message(&value), None)`
- 入力 3: `{ "error": { "message": null } }` → priority1 None (no key) + priority2 (object) + priority3 nested null → None
- assert: `assert_eq!(super::extract_error_message(&value), None)`
- 入力 4: `{ "message": 1.5, "error": [1, 2], "extra": "noise" }` → priority1 number + priority2 array + priority3 None → None
- assert: `assert_eq!(super::extract_error_message(&value), None)`
- 意図: 全 priority が non-string な場合の終端 None 返却を 4 シナリオで網羅。fallback chain が「最終的に None になる」総合契約を保護。
