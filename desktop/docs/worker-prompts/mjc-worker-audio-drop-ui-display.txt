### 変更 3. JSX に drop count 表示を追加 (line 2348-2357 付近)

既存の `audioLevelListenerError` 表示ブロック (line 2348-2357) の **直後** に対称な構造で 2 つのブロックを追加する:

```tsx
{audioDropCountListenerError && (
  <p
    className="meeting-error meeting-alert"
    role="alert"
    aria-label={`音声 drop 監視エラー: ${audioDropCountListenerError}`}
    title={`音声 drop 監視エラー: ${audioDropCountListenerError}`}
  >
    {audioDropCountListenerError}
  </p>
)}
{(microphoneDropCountTotal > 0 || systemAudioDropCountTotal > 0) && (
  <p
    className="meeting-error meeting-alert"
    role="status"
    aria-live="polite"
    aria-atomic="true"
    aria-label={`音声サンプル破棄: マイク ${microphoneDropCountTotal} サンプル, システム音声 ${systemAudioDropCountTotal} サンプル`}
    title={`音声サンプル破棄: マイク ${microphoneDropCountTotal} サンプル, システム音声 ${systemAudioDropCountTotal} サンプル`}
  >
    音声サンプル破棄: マイク {microphoneDropCountTotal} サンプル / システム音声 {systemAudioDropCountTotal} サンプル
  </p>
)}
```

- 1 つ目は `audioDropCountListenerError` (listener エラー) 表示。`role="alert"` で `audioLevelListenerError` ブロックと完全対称。
- 2 つ目は累積 drop count 表示。**両方が 0 なら何も表示しない**条件 `microphoneDropCountTotal > 0 || systemAudioDropCountTotal > 0` で UI ノイズを最小化。`role="status"` + `aria-live="polite"` で screen reader への過度な割り込みを避ける (alert は本質的に urgent な error 用)。
- className `meeting-error meeting-alert` は既存 audio-level error 表示と同じ視覚 style 流用。新しい css class は追加しない。

# 実装上の注意 (踏みやすい罠)

- **3 つの state 値の参照が UI へ移ったことを必ず確認する**。tsc `noUnusedLocals` は state 変数自体が read されないと error を出す。useState の **2 個目要素 (setter) は handler 内で read 済み** だが、**1 個目要素 (state value) は本ループで必ず JSX 内で 1 回以上 read** する必要がある。変更 3 で 3 つすべて (microphoneDropCountTotal / systemAudioDropCountTotal / audioDropCountListenerError) を JSX 内で参照するので OK。
- **eslint-disable 削除は変更 1 と必ずペアで行う**。変更 1 を適用せず変更 2 だけ行うと exhaustive-deps lint が再発する。**両方同時にコミット可能な状態でファイルを保存する**こと。
- **既存 audio-level error 表示ブロック (line 2348-2357) は変更しない**。直後に追加するだけ。
- **drop count 表示の文言**: 「音声サンプル破棄: マイク N サンプル / システム音声 M サンプル」に統一。会話的な日本語で短く、aria-label と本文で同じ表現。
- **両方 0 の場合は表示しない**: 通常の録音時は drop=0 なので、UI ノイズを避けるためゼロ条件で hide する。これは UX 設計判断 = 「異常時のみ可視化」原則に沿う。
- **className `meeting-error meeting-alert` を流用**: 新しい css class は追加しない (本ループは pure logic 追加、style 設計は別ループ)。視覚的に既存の error と同じ見た目になる = drop は録音の異常状態であるという情報設計。
- **conditional render 順番**: console.warn → audioDropCountListenerError 表示 → drop count 表示 の順で読みやすさ優先。listener error が出ているときは drop count も出る可能性があるが、両者独立で表示できる構造。
