## 未保護の軸 (本 Loop で 3 軸 application)

### T1 (Debug 出力 = 型名 + 全 field 名 + Some/None 表現 + enum variant 名軸)
名称: `transcription_error_payload_debug_output_contains_struct_name_field_names_and_some_variant_with_enum_name`
契約:
- `TranscriptionErrorPayload { error: "msg".to_string(), source: Some(TranscriptionSource::SystemAudio) }` の `format!("{:?}", payload)` が以下を全て含むこと:
  - 型名 `"TranscriptionErrorPayload"`
  - field 名 `"error"`, `"source"`
  - 値 `"msg"`
  - Some の Debug 表現 `"Some"` (Option::Some の Debug 出力)
  - enum variant 名 `"SystemAudio"` (TranscriptionSource::SystemAudio の Debug 出力)
- 別 case で `source: None` の payload を生成し、その Debug 出力に `"None"` が含まれることも追加 assert
- `#[derive(Debug)]` 自動派生による「struct with Option<enum> field の Debug 出力」契約を executable specification 化
- 「将来 source field の Debug 表現を変える誤改修」「TranscriptionSource enum variant rename の波及」を遮断する装置

### T2 (Clone 後 Debug 出力一致軸)
名称: `transcription_error_payload_debug_output_equals_after_clone_for_some_and_none_variants`
契約:
- 元 TranscriptionErrorPayload (Some 持ち) と `.clone()` 後の Debug 出力が完全一致 (`assert_eq!`)
- 元 TranscriptionErrorPayload (None 持ち) と `.clone()` 後の Debug 出力が完全一致 (`assert_eq!`)
- `#[derive(Debug, Clone)]` の組み合わせで「Clone が field 順や Debug 表現まで含めて 100% 同一」契約の executable specification 化
- 「将来 Clone を手動実装で Option field を加工する誤改修」を遮断する装置

### T3 (PartialEq の executable specification 軸 = reflexive + 異 field で 不等)
名称: `transcription_error_payload_partial_eq_holds_reflexive_and_differs_for_distinct_error_or_source`
契約:
- 同 (error, source) を持つ 2 つの payload で `==` (reflexive 含む)
- 異 error / 同 source で `!=`
- 同 error / 異 source (Some(Microphone) vs Some(SystemAudio)) で `!=`
- 同 error / Some vs None の source 差で `!=`
- `#[derive(PartialEq, Eq)]` による「全 field が等値判定対象」契約の executable specification 化
- 「将来 PartialEq を手動実装で source を等値判定から除外する誤改修」を遮断する装置
