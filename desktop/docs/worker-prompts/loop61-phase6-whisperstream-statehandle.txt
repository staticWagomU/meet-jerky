注: `<COMMIT_HASH>` は **実コミット後の hash** で確実に置換すること。手順例:
1. すべての変更を `git add`
2. `git commit -m "<commit メッセージ>"` で commit
3. `hash=$(git rev-parse --short HEAD)`
4. `sed -i.bak "s/<COMMIT_HASH>/$hash/" AGENT_LOG.md && rm AGENT_LOG.md.bak`
5. `git add AGENT_LOG.md && git commit --amend --no-edit`
6. `--amend` 後の hash を確認し、もし AGENT_LOG.md の hash が `--amend` で変わった hash と異なるなら、もう 1 回置換 + amend

3. `git commit -m "..."` で commit
4. 直後に `hash=$(git rev-parse --short HEAD)` を取得し、AGENT_LOG.md に追記
5. `git add AGENT_LOG.md && git commit --amend --no-edit` で amend
6. 最終 `git rev-parse --short HEAD` の hash を AGENT_LOG.md と再度合わせる

**最も安全な方法**: AGENT_LOG.md の `## commit` セクションを **2 commit pattern (= refactor commit + chore commit hash 反映)** で書く。
